    C++ 语言教程：指针

    原文链接：cplusplus.com - Pointers

    本文为 cplusplus.com 指针教程的中文翻译版，保留了原文的所有代码示例和示意图。

    指针

    在之前的章节中，变量被解释为计算机内存中的位置，可以通过其标识符（即变量名）来访问。这样一来，程序就不需要关心数据在内存中的物理地址；它只需在需要引用变量时使用标识符即可。

    对于 C++ 程序而言，计算机的内存就像一系列连续的内存单元，每个单元大小为一个字节，且每个单元都有一个唯一的地址。这些单字节内存单元按一定顺序排列，使得大于一个字节的数据表示可以占据具有连续地址的内存单元。

    这样一来，每个单元都可以通过其唯一地址在内存中轻松定位。例如，地址为 1776 的内存单元总是紧跟在地址为 1775 的单元之后，且在地址为 1777 的单元之前，它恰好比地址 776 的单元多一千个位置，比地址 2776 的单元少一千个位置。

    当声明一个变量时，存储其值所需的内存会被分配到内存中的特定位置（即其内存地址）。通常，C++ 程序不会主动决定其变量存储的确切内存地址。幸运的是，这项任务交给了程序运行的环境——通常是操作系统，它在运行时决定具体的内存位置。然而，程序在运行时能够获取变量的地址可能是有用的，以便访问相对于该变量位于特定位置的数据单元。

    取地址运算符（&）

    变量的地址可以通过在变量名前加上与号（&）来获取，&被称为取地址运算符（address-of operator）。例如：

    foo = &myvar;

    这会将变量 myvar 的地址赋给 foo；通过在变量名 myvar 前加上取地址运算符（&），我们不再将变量本身的内容赋给 foo，而是将其地址赋给 foo。

    变量在内存中的实际地址在运行前是无法知道的，但为了帮助澄清一些概念，我们假设 myvar 在运行时被放置在内存地址 1776 处。

    在这种情况下，考虑以下代码片段：

    myvar = 25;
foo = &myvar;
bar = myvar;

    执行上述代码后，各变量中包含的值如下图所示：

    首先，我们将值 25 赋给了 myvar（一个我们假设其内存地址为 1776 的变量）。

    第二条语句将 myvar 的地址赋给 foo，我们假设该地址为 1776。

    最后，第三条语句将 myvar 中包含的值赋给 bar。这是一个标准的赋值操作，正如之前章节中多次做过的那样。

    第二条和第三条语句的主要区别在于取地址运算符（&）的出现。

    存储另一个变量地址的变量（如上例中的 foo）在 C++ 中被称为指针。指针是语言中一个非常强大的特性，在底层编程中有许多用途。稍后我们将看到如何声明和使用指针。

    解引用运算符（*）

    如刚才所见，存储另一个变量地址的变量被称为指针。指针被称为"指向"其存储地址的变量。

    指针的一个有趣属性是，它们可以直接访问所指向的变量。这通过在指针名前加上解引用运算符（*）来实现。该运算符本身可以读作"所指向的值"。

    因此，沿用前例的值，以下语句：

    baz = *foo;

    可以读作："baz 等于 foo 所指向的值"，该语句实际上会将值 25 赋给 baz，因为 foo 的值是 1776，而 1776 所指向的值（按照上面的例子）是 25。

    重要的是要清楚区分：foo 指的是值 1776，而 *foo（在标识符前加星号 *）指的是存储在地址 1776 处的值，在本例中是 25。请注意包含或不包含解引用运算符的区别（我添加了说明性注释，解释这两个表达式各自的含义）：

    baz = foo;   // baz 等于 foo 的值（1776）
baz = *foo;  // baz 等于 foo 所指向的值（25）

    因此，取地址运算符和解引用运算符是互补的：

    • & 是取地址运算符，可以简单读作"……的地址"
    • * 是解引用运算符，可以读作"……所指向的值"

    因此，它们具有某种相反的含义：用 & 获取的地址可以用 * 来解引用。

    之前，我们执行了以下两个赋值操作：

    myvar = 25;
foo = &myvar;

    在这两条语句之后，以下所有表达式都会给出 true 的结果：

    myvar == 25
&myvar == 1776
foo == 1776
*foo == 25

    第一个表达式很清楚，考虑到对 myvar 执行的赋值操作是 myvar = 25。第二个使用了取地址运算符（&），它返回 myvar 的地址，我们假设其值为 1776。第三个比较明显，因为第二个表达式为真，而对 foo 执行的赋值操作是 foo = &myvar。第四个表达式使用了解引用运算符（*），可以读作"所指向的值"，而 foo 所指向的值确实是 25。

    所以，经过以上分析，你还可以推断出，只要 foo 指向的地址保持不变，以下表达式也将为真：

    *foo == myvar

    声明指针

    由于指针能够直接引用它所指向的值，当指针指向 char 时与指向 int 或 float 时具有不同的属性。一旦解引用，类型需要被知道。为此，指针的声明需要包含指针将要指向的数据类型。

    指针的声明遵循以下语法：

    type * name;

    其中 type 是指针所指向的数据类型。这个类型不是指针本身的类型，而是指针所指向数据的类型。例如：

    int * number;
char * character;
double * decimals;

    这是三个指针的声明。每一个都旨在指向不同的数据类型，但事实上，它们都是指针，并且它们在内存中很可能占据相同大小的空间（指针在内存中的大小取决于程序运行的平台）。然而，它们所指向的数据不占据相同大小的空间，也不属于相同的类型：第一个指向 int，第二个指向 char，最后一个指向 double。因此，虽然这三个示例变量都是指针，但它们实际上具有不同的类型：int*、char* 和 double*，取决于它们所指向的类型。

    请注意，声明指针时使用的星号（*）仅表示它是一个指针（它是其类型复合说明符的一部分），不应与前面看到的解引用运算符混淆，虽然解引用运算符也用星号（*）书写。它们只是用相同符号表示的两个不同事物。

    让我们看一个指针的例子：

    // my first pointer
#include <iostream>
using namespace std;

int main ()
{
  int firstvalue, secondvalue;
  int * mypointer;

  mypointer = &firstvalue;
  *mypointer = 10;
  mypointer = &secondvalue;
  *mypointer = 20;

  cout << "firstvalue is " << firstvalue << '\n';
  cout << "secondvalue is " << secondvalue << '\n';
  return 0;
}

    输出：

    firstvalue is 10
secondvalue is 20

    请注意，虽然 firstvalue 和 secondvalue 在程序中都没有被直接设置任何值，但两者最终都通过使用 mypointer 间接地获得了值。过程如下：

    首先，使用取地址运算符（&）将 firstvalue 的地址赋给 mypointer。然后，将 mypointer 所指向的值赋为 10。因为此时 mypointer 正指向 firstvalue 的内存位置，这实际上修改了 firstvalue 的值。

    为了演示指针在程序的生命周期中可以指向不同的变量，该示例用 secondvalue 和同一个指针 mypointer 重复了这一过程。

    这里有一个稍微更复杂的例子：

    // more pointers
#include <iostream>
using namespace std;

int main ()
{
  int firstvalue = 5, secondvalue = 15;
  int * p1, * p2;

  p1 = &firstvalue;  // p1 = firstvalue 的地址
  p2 = &secondvalue; // p2 = secondvalue 的地址
  *p1 = 10;          // p1 所指向的值 = 10
  *p2 = *p1;         // p2 所指向的值 = p1 所指向的值
  p1 = p2;           // p1 = p2（指针的值被复制）
  *p1 = 20;          // p1 所指向的值 = 20

  cout << "firstvalue is " << firstvalue << '\n';
  cout << "secondvalue is " << secondvalue << '\n';
  return 0;
}

    输出：

    firstvalue is 10
secondvalue is 20

    每个赋值操作都包含一条注释，说明每行可以如何解读：即，将与号（&）替换为"……的地址"，将星号（*）替换为"……所指向的值"。

    请注意，有些表达式使用了指针 p1 和 p2，有的带解引用运算符（*），有的不带。使用解引用运算符（*）的表达式的含义与不使用的截然不同。当此运算符出现在指针名之前时，表达式指的是被指向的值；而当指针名不带此运算符出现时，它指的是指针本身的值（即指针所指向内容的地址）。

    另一件可能引起你注意的事情是这一行：

    int * p1, * p2;

    这声明了上例中使用的两个指针。但请注意，每个指针都有一个星号（*），以便两者都具有类型 int*（指向 int 的指针）。这是由于优先级规则所要求的。注意，如果代码改为：

    int * p1, p2;

    p1 的类型确实是 int*，但 p2 的类型将是 int。空格对此毫无影响。但无论如何，只需记住每个指针放一个星号，就足以满足大多数想要在一条语句中声明多个指针的用户的需求。或者更好的做法是：为每个变量使用单独的声明语句。

    指针与数组

    数组的概念与指针的概念密切相关。事实上，数组的工作方式非常类似于指向其第一个元素的指针，而且数组总是可以隐式转换为适当类型的指针。例如，考虑以下两个声明：

    int myarray [20];
int * mypointer;

    以下赋值操作是有效的：

    mypointer = myarray;

    此后，mypointer 和 myarray 将是等价的，并具有非常相似的属性。主要区别在于 mypointer 可以被赋予不同的地址，而 myarray 永远不能被赋予任何值，它将始终代表同一块 20 个 int 类型元素。因此，以下赋值是无效的：

    myarray = mypointer;  // 无效！

    让我们看一个混合使用数组和指针的例子：

    // more pointers
#include <iostream>
using namespace std;

int main ()
{
  int numbers[5];
  int * p;

  p = numbers;  *p = 10;
  p++;  *p = 20;
  p = &numbers[2];  *p = 30;
  p = numbers + 3;  *p = 40;
  p = numbers;  *(p+4) = 50;

  for (int n=0; n<5; n++)
    cout << numbers[n] << ", ";
  return 0;
}

    输出：

    10, 20, 30, 40, 50,

    指针和数组支持相同的操作集合，对两者具有相同的含义。主要区别在于指针可以被赋予新的地址，而数组不能。

    在关于数组的章节中，方括号（[]）被解释为指定数组元素的索引。事实上，这些方括号是一种解引用运算符，称为偏移运算符（offset operator）。它们对所跟随的变量进行解引用，就像 * 一样，但它们还将方括号中的数字加到被解引用的地址上。例如：

    a[5] = 0;       // a [偏移量为 5] = 0
*(a+5) = 0;     // (a+5) 所指向的值 = 0

    这两个表达式是等价且有效的，不仅当 a 是指针时如此，当 a 是数组时也是如此。请记住，如果是数组，其名称可以像指向其第一个元素的指针一样使用。

    指针初始化

    指针可以在定义时就初始化为指向特定位置：

    int myvar;
int * myptr = &myvar;

    这段代码执行后变量的状态与以下代码相同：

    int myvar;
int * myptr;
myptr = &myvar;

    当指针被初始化时，被初始化的是它们指向的地址（即 myptr），而不是被指向的值（即 *myptr）。因此，上面的代码不应与以下代码混淆：

    int myvar;
int * myptr;
*myptr = &myvar;   // 无效代码！

    这无论如何都不太有意义（而且不是有效的代码）。

    指针声明中的星号（*）（第 2 行）仅表示它是一个指针，它不是解引用运算符（如第 3 行中的那样）。这两件事只是碰巧使用了相同的符号：*。一如既往，空格是无关紧要的，永远不会改变表达式的含义。

    指针可以初始化为变量的地址（如上面的例子），也可以初始化为另一个指针（或数组）的值：

    int myvar;
int *foo = &myvar;
int *bar = foo;

    指针算术

    对指针进行算术运算与对常规整数类型的运算略有不同。首先，只允许加法和减法运算；其他运算在指针的世界中没有意义。但加法和减法对指针的行为略有不同，取决于它们所指向的数据类型的大小。

    当介绍基本数据类型时，我们看到不同类型具有不同的大小。例如：char 的大小始终为 1 字节，short 通常大于此值，而 int 和 long 更大；这些的确切大小取决于系统。例如，让我们假设在某个系统中，char 占 1 字节，short 占 2 字节，long 占 4 字节。

    假设现在我们在该编译器中定义三个指针：

    char *mychar;
short *myshort;
long *mylong;

    并且我们知道它们分别指向内存位置 1000、2000 和 3000。

    因此，如果我们写：

    ++mychar;
++myshort;
++mylong;

    mychar 如预期将包含值 1001。但不太明显的是，myshort 将包含值 2002，mylong 将包含 3004，尽管它们各自只被递增了一次。原因是，当对指针加 1 时，指针被设置为指向同类型的下一个元素，因此，它所指向类型的字节大小被加到了指针上。

    这适用于对指针进行任何数字的加法和减法。如果我们写以下代码，情况完全相同：

    mychar = mychar + 1;
myshort = myshort + 1;
mylong = mylong + 1;

    关于递增（++）和递减（--）运算符，它们都可以作为表达式的前缀或后缀使用，行为略有不同：作为前缀时，递增在表达式求值之前发生；作为后缀时，递增在表达式求值之后发生。这也适用于递增和递减指针的表达式，它们可以成为包含解引用运算符（*）的更复杂表达式的一部分。回顾运算符优先级规则，我们可以回忆，后缀运算符（如递增和递减）的优先级高于前缀运算符（如解引用运算符 *）。因此，以下表达式：

    *p++

    等价于 *(p++)。它的作用是增加 p 的值（使其指向下一个元素），但因为 ++ 作为后缀使用，整个表达式被求值为指针原来指向的值（即递增之前它所指向的地址处的值）。

    本质上，以下是解引用运算符与递增运算符的前缀和后缀版本的四种可能组合（同样适用于递减运算符）：

    *p++   // 等同于 *(p++)：递增指针，并解引用未递增的地址
*++p   // 等同于 *(++p)：递增指针，并解引用已递增的地址
++*p   // 等同于 ++(*p)：解引用指针，并递增其指向的值
(*p)++ // 解引用指针，并后递增其指向的值

    一个典型但不太简单的涉及这些运算符的语句是：

    *p++ = *q++;

    因为 ++ 的优先级高于 *，p 和 q 都被递增了，但因为两个递增运算符（++）都作为后缀而非前缀使用，赋给 *p 的值是 *q 在 p 和 q 都被递增之前的值。然后两者都被递增。它大致等价于：

    *p = *q;
++p;
++q;

    一如既往，括号可以通过增加表达式的可读性来减少混淆。

    指针与 const

    指针可以通过地址访问变量，这种访问可能包括修改所指向的值。但也可以声明只能读取所指向值而不能修改它的指针。为此，只需将指针所指向的类型限定为 const。例如：

    int x;
int y = 10;
const int * p = &y;
x = *p;          // 正确：读取 p
*p = x;          // 错误：修改 p，但 p 是 const 限定的

    这里 p 指向一个变量，但以 const 限定的方式指向它，意味着它可以读取所指向的值，但不能修改它。还要注意，表达式 &y 的类型是 int*，但它被赋给了一个类型为 const int* 的指针。这是允许的：指向非 const 的指针可以隐式转换为指向 const 的指针。但反过来不行！作为安全特性，指向 const 的指针不能隐式转换为指向非 const 的指针。

    指向 const 元素的指针的一个用例是作为函数参数：接受指向非 const 的指针作为参数的函数可以修改作为参数传递的值，而接受指向 const 的指针作为参数的函数则不能。

    // pointers as arguments:
#include <iostream>
using namespace std;

void increment_all (int* start, int* stop)
{
  int * current = start;
  while (current != stop) {
    ++(*current);  // 递增所指向的值
    ++current;     // 递增指针
  }
}

void print_all (const int* start, const int* stop)
{
  const int * current = start;
  while (current != stop) {
    cout << *current << '\n';
    ++current;     // 递增指针
  }
}

int main ()
{
  int numbers[] = {10,20,30};
  increment_all (numbers, numbers+3);
  print_all (numbers, numbers+3);
  return 0;
}

    输出：

    11
21
31

    请注意，print_all 使用了指向常量元素的指针。这些指针指向它们不能修改的常量内容，但它们本身不是常量：即，指针仍然可以被递增或赋予不同的地址，尽管它们不能修改所指向的内容。

    这就是指针的 const 性增加了第二个维度的地方：指针本身也可以是 const。这通过在所指向的类型之后（星号之后）追加 const 来指定：

    int x;
      int *       p1 = &x;  // 非 const 指针指向非 const int
const int *       p2 = &x;  // 非 const 指针指向 const int
      int * const p3 = &x;  // const 指针指向非 const int
const int * const p4 = &x;  // const 指针指向 const int

    const 与指针的语法确实很棘手，识别最适合每种用途的情况通常需要一些经验。无论如何，尽早正确理解指针（和引用）的 const 性很重要，但如果这是你第一次接触 const 和指针的混合，也不必太担心无法掌握所有内容。更多用例将在后续章节中出现。

    为了给 const 与指针的语法增加一点更多的混淆，const 限定符可以放在所指向类型之前或之后，含义完全相同：

    const int * p2a = &x;  //      非 const 指针指向 const int
int const * p2b = &x;  // 也是 非 const 指针指向 const int

    与星号周围的空格一样，在这种情况下 const 的顺序仅仅是风格问题。本章使用前缀 const，因为出于历史原因这似乎更普遍，但两者完全等价。每种风格的优缺点在互联网上仍然被激烈争论。

    指针与字符串字面量

    如前所述，字符串字面量是包含以空字符结尾的字符序列的数组。在之前的章节中，字符串字面量被用于直接插入 cout、初始化字符串和初始化字符数组。

    但它们也可以被直接访问。字符串字面量是适当数组类型的数组，包含其所有字符加上终止空字符，每个元素的类型为 const char（作为字面量，它们永远不能被修改）。例如：

    const char * foo = "hello";

    指针 foo 指向一个字符序列。因为指针和数组在表达式中的行为基本相同，foo 可以像以空字符结尾的字符序列数组那样用来访问字符。例如：

    void 指针

    void 类型指针是一种特殊类型的指针。在 C++ 中，void 表示类型的缺失。因此，void 指针是指向没有类型的值的指针（因此也具有不确定的长度和不确定的解引用属性）。

    这赋予了 void 指针极大的灵活性，能够指向任何数据类型，从整数值或浮点数到字符串。作为代价，它们有一个很大的限制：它们指向的数据不能被直接解引用（这是合乎逻辑的，因为我们没有类型可以解引用到），因此，void 指针中的任何地址需要先转换为指向具体数据类型的其他指针类型，然后才能被解引用。

    它的一种可能用途是将通用参数传递给函数。例如：

    // increaser
#include <iostream>
using namespace std;

void increase (void* data, int psize)
{
  if ( psize == sizeof(char) )
  { char* pchar; pchar=(char*)data; ++(*pchar); }
  else if (psize == sizeof(int) )
  { int* pint; pint=(int*)data; ++(*pint); }
}

int main ()
{
  char a = 'x';
  int b = 1602;
  increase (&a, sizeof(a));
  increase (&b, sizeof(b));
  cout << a << ", " << b << '\n';
  return 0;
}

    输出：

    y, 1603

    sizeof 是 C++ 语言中内置的运算符，返回其参数的字节大小。对于非动态数据类型，此值是常量。因此，例如 sizeof(char) 是 1，因为 char 的大小始终为一个字节。

    无效指针和空指针

    原则上，指针旨在指向有效地址，例如变量的地址或数组中元素的地址。但指针实际上可以指向任何地址，包括不引用任何有效元素的地址。典型的例子包括未初始化的指针和指向数组中不存在元素的指针：

    int * p;               // 未初始化的指针（局部变量）
int myarray[10];
int * q = myarray+20;  // 越界元素

    p 和 q 都不指向已知包含值的地址，但以上语句都不会导致错误。在 C++ 中，指针可以取任何地址值，无论该地址是否确实有内容。可能导致错误的是解引用这样的指针（即实际访问它们指向的值）。访问这样的指针会导致未定义行为，从运行时错误到访问某个随机值。

    但是，有时指针确实需要明确地指向"无处"，而不是仅仅指向一个无效地址。对于这种情况，存在一个任何指针类型都可以取的特殊值：空指针值。这个值在 C++ 中可以用两种方式表达：用整数值零，或用 nullptr 关键字：

    int * p = 0;
int * q = nullptr;

    这里，p 和 q 都是空指针，意味着它们明确地指向无处，而且它们实际上比较相等：所有空指针与其他空指针比较相等。在旧代码中也很常见到使用定义的常量 NULL 来引用空指针值：

    int * r = NULL;

    NULL 在标准库的多个头文件中定义，并被定义为某个空指针常量值的别名（如 0 或 nullptr）。

    不要混淆空指针与 void 指针！ 空指针是任何指针都可以取的一个值，用来表示它指向"无处"，而 void 指针是一种可以指向某处但没有特定类型的指针类型。一个指的是存储在指针中的值，另一个指的是它所指向的数据类型。

    指向函数的指针

    C++ 允许对函数指针进行操作。典型的用途是将一个函数作为参数传递给另一个函数。函数指针的声明语法与常规函数声明相同，只是函数名被括在括号 () 中，并在名称前插入一个星号（*）：

    // pointer to functions
#include <iostream>
using namespace std;

int addition (int a, int b)
{ return (a+b); }

int subtraction (int a, int b)
{ return (a-b); }

int operation (int x, int y, int (*functocall)(int,int))
{
  int g;
  g = (*functocall)(x,y);
  return (g);
}

int main ()
{
  int m,n;
  int (*minus)(int,int) = subtraction;

  m = operation (7, 5, addition);
  n = operation (20, m, minus);

  cout << n;
  return 0;
}

    输出：

    8

    在上面的例子中，minus 是一个指向具有两个 int 类型参数的函数的指针。它被直接初始化为指向函数 subtraction：

    int (* minus)(int,int) = subtraction;
