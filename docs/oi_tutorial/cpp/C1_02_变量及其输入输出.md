## 三、教学重点与难点

### 1\. 教学重点

-   变量命名规则：字母/数字/下划线、不能数字开头、不能是关键字、区分大小写。
-   四种基本类型 `int`/`long long`、`float`/`double`、`char`、`bool` 各自存放什么、用什么写法给值。
-   变量定义语法 `类型 变量名;`，赋值 `=`，初始化 `int a = 10;`。
-   cin 输入语法 `cin >> 变量;`，与 cout 方向相反（`>>` 向右流入变量）。
-   算术运算五则，尤其整数除法 `7/2=3` 与求余 `7%2=1`。

### 2\. 教学难点

-   理解"变量是装数据的盒子"，类型决定盒子能装什么。
-   `=` 是赋值（把右边的值放进左边的变量），不是数学里的"等于"。
-   整数除整数得整数（直接丢小数，不是四舍五入），是本课最大坑。
-   `%` 求余只能用于整数，不能用于 double。
-   cin 的 `>>` 与 cout 的 `<<` 方向容易记混。

## 四、为什么需要变量

### 1\. 第一课的局限

第一课我们用 cout 输出的是"写死"的内容：

```cpp
cout << "3 + 4 = " << 3 + 4 << endl;   // 永远只算 3+4

```

如果题目要算 `5 + 8`、`100 + 200`，难道每次都改代码？当然不行。我们需要一种东西，能**在程序运行时临时存放数据**，还能反复使用、反复计算——这就是**变量**。

### 2\. 变量带来的改变

有了变量，程序可以"先读入数据，再计算，最后输出"：

```cpp
int a, b;            // 准备两个"盒子"装整数
cin >> a >> b;       // 从键盘读入 a 和 b
cout << a + b;       // 输出它们的和

```

这样一份代码，输入 `3 4` 就输出 `7`，输入 `100 200` 就输出 `300`——程序变得通用了。这就是变量的价值。

## 五、变量是什么

**变量是内存里的一块空间，用来存放数据。** 你可以把它想象成一个"贴了标签的盒子"：

-   **盒子** = 内存里的一块空间
-   **标签（变量名）** = 你给这块空间起的名字
-   **内容（变量的值）** = 盒子里装的数据
-   **盒子的形状（数据类型）** = 决定能装什么类型的数据

```
┌─────────────┐
│   a = 10    │   ← 变量名 a，值是 10，类型 int（装整数）
└─────────────┘

```

关键特点：

-   变量的值**可以改变**（所以叫"变"量）：`a = 10;` 之后可以 `a = 20;`，新值覆盖旧值。
-   变量**必须先定义再使用**：没定义过的变量名不能用。

## 六、变量名规范与注意事项

### 1\. 命名规则（必须遵守，否则编译错误）

| 规则 | 说明 | 合法示例 | 非法示例 |
| --- | --- | --- | --- |
| 只能由字母、数字、下划线组成 | 不能有空格、不能有其他符号 | `score`、`a1`、`_tmp` | `s core`、`a-b`、`a#` |
| 不能以数字开头 | 第一个字符必须是字母或下划线 | `a1`、`_x` | `1a`、`2nd` |
| 不能是 C++ 关键字 | 关键字是语言保留的词 | `num`、`sum` | `int`、`return`、`main` |
| 区分大小写 | `a` 和 `A` 是两个不同变量 | `a` 与 `A` 不同 | — |

### 2\. 常见 C++ 关键字（先记这几个，别拿来当变量名）

`int`、`double`、`char`、`bool`、`return`、`if`、`else`、`for`、`while`、`main`（main 不是关键字但保留作主函数名，别用）。

### 3\. 命名建议（好习惯，不强制）

-   **见名知意**：`score` 比 `s` 好，`age` 比 `a` 好。竞赛题变量多时可短，如 `a`、`b`、`n`。
-   **风格统一**：要么用驼峰 `studentScore`，要么用下划线 `student_score`，别混用。
-   **别太长**：`the_score_of_the_student` 太啰嗦，`score` 就够。

### 4\. 注意事项

-   变量名不能重复定义（同一作用域内）：`int a; int a;` 会报错。
-   不要用拼音缩写到自己也看不懂：`cj` 是成绩还是差价？写 `score` 更清楚。
-   大小写敏感是双刃剑：`Sum` 和 `sum` 是两个变量，别手滑。

## 七、数据类型与四种基本类型

### 1\. 为什么要有类型

不同数据占的空间不同、能做的运算不同。比如：

-   整数 `3` 和 `4` 能做 `3 % 4`（求余）；
-   小数 `3.14` 不能做 `%`；
-   字符 `'A'` 本质存的是 ASCII 码（整数），所以也能做加减运算（如 `'a'-32` 转大写）。

所以定义变量时，必须**先告诉计算机这个变量是什么类型**。

### 2\. 四种基本数据类型

| 类型 | 关键字 | 存放内容 | 占用空间 | 取值范围（参考） | 给值写法 |
| --- | --- | --- | --- | --- | --- |
| 整型 | `int` | 整数 | 4 字节 | 约 ±21 亿（−2147483648 ~ 2147483647） | `10`、`-5` |
| 整型 | `long long` | 整数 | 8 字节 | 约 ±9.2×10¹⁸（−9223372036854775808 ~ 9223372036854775807） | `1000000000`、`5LL` |
| 浮点型 | `float` | 小数 | 4 字节 | 约 ±3.4×10³⁸，有效数字 6~7 位 | `3.14f`、`-0.5f` |
| 浮点型 | `double` | 小数 | 8 字节 | 约 ±1.7×10³⁰⁸，有效数字 15~16 位 | `3.14`、`-0.5` |
| 字符型 | `char` | 单个字符 | 1 字节 | 实际存 ASCII 码（0~127） | `'A'`、`'9'` |
| 布尔型 | `bool` | 真或假 | 1 字节 | `true`(1) / `false`(0) | `true`、`false` |

### 3\. 各类型详解

**① 整型 `int` 与 `long long`**

-   `int`：存整数，4 字节，范围约 ±21 亿，是最常用的整型。
-   `long long`：存整数，8 字节，范围约 ±9.2×10¹⁸。当数据或结果可能超过 21 亿时（如人数、大数求和），必须用 `long long`，否则会溢出得到错误结果。
-   写法：直接写数字，不加引号；`long long` 字面量可加 `LL` 后缀。

```cpp
int age = 12;
int score = 100;
long long big = 1000000000000LL;   // 超过 21 亿，用 long long

```

**② 浮点型 `float` 与 `double`**

-   `float`：存小数，4 字节，有效数字 6~7 位，精度较低。
-   `double`：存小数，8 字节，有效数字 15~16 位，精度高。
-   信奥里**小数一律用 `double`**，不用 `float`（精度不够容易出错）。
-   注意：`double` 存小数有精度误差，`0.1 + 0.2` 可能不精确等于 `0.3`，后续会讲。

```cpp
float f = 3.14f;        // float 字面量加 f 后缀
double pi = 3.14159;
double temp = 36.5;

```

**③ 字符型 `char`**

-   存**单个**字符，用**单引号** `' '` 括起来。
-   字符本质存的是 ASCII 码：`'A'` 实际存的是 65，`'a'` 是 97，`'0'` 是 48。
-   单引号里只能放一个字符：`'A'` 对，`'AB'` 错。

```cpp
char ch = 'A';
char op = '+';

```

**④ 布尔型 `bool`**

-   只有两个值：`true`（真，等于 1）和 `false`（假，等于 0）。
-   常用来表示"是/否"，如"是否及格"。

```cpp
bool pass = true;
bool isEnd = false;

```

### 4\. 类型选择口诀

**整数用 int（可能超 21 亿用 long long），小数用 double，单个字符用 char，是非判断用 bool。**

## 八、变量的定义、赋值与初始化

### 1\. 定义变量

语法：`类型 变量名;`

```cpp
int a;            // 定义一个整型变量 a
double x;         // 定义一个浮点变量 x
char ch;          // 定义一个字符变量 ch
bool flag;        // 定义一个布尔变量 flag

```

可以一次定义多个同类型变量，用逗号隔开：

```cpp
int a, b, c;      // 一次定义三个整型变量

```

### 2\. 赋值（=）

`=` 是**赋值运算符**：把右边的值放进左边的变量。

```cpp
a = 10;           // 把 10 放进 a
a = 20;           // 把 a 的值改成 20（旧值 10 被覆盖）
b = a;            // 把 a 的值复制给 b（此时 b 也是 20）

```

> **关键理解：** `=` 不是"等于"，是"把右边的放进左边"。所以 `a = a + 1;` 是合法的——意思是"把 a 加 1 后再放回 a"。

### 3\. 初始化（定义时直接给值）

定义的同时给值，叫初始化：

```cpp
int a = 10;            // 定义 a 并初始化为 10
double pi = 3.14;      // 定义 pi 并初始化为 3.14
char ch = 'A';         // 定义 ch 并初始化为 'A'
int b = 3, c = 4;      // 定义多个并分别初始化

```

> **重要：** 变量**未赋值就读取**，得到的是"垃圾值"（内存里残留的随机数）。养成"定义即初始化"的好习惯。

### 4\. 三种写法对照

| 写法 | 含义 | 推荐度 |
| --- | --- | --- |
| `int a;` | 只定义，未给值 | 易出垃圾值 |
| `int a = 10;` | 定义并初始化 | 推荐 |
| `int a; a = 10;` | 先定义，再赋值 | 可以，分两步 |

## 九、用 cin 输入变量的值

### 1\. 基本语法

cin 是"标准输入流"对象，`>>` 叫**提取运算符**。

**读法：** 把 `>>` 左边 cin（键盘）的数据，提取到右边的变量里。

```cpp
cin >> 变量名;

```

注意方向：`>>` 向右，数据从键盘流向变量。

### 2\. 输入一个变量

```cpp
int a;
cin >> a;         // 从键盘读入一个整数放进 a

```

运行时在键盘输入 `10` 回车，a 就变成 10。

### 3\. 输入多个变量

用多个 `>>` 串起来，输入时用空格或回车分隔：

```cpp
int a, b;
cin >> a >> b;    // 读入两个整数

```

输入 `3 4` 或 `3↵4`（↵表示回车）都可以，a 得 3，b 得 4。

### 4\. 不同类型的输入

```cpp
int n;  cin >> n;        // 输入整数：10
double x; cin >> x;      // 输入小数：3.14
char ch; cin >> ch;      // 输入单个字符：A

```

> **注意：** `cin >> ch` 读字符时会**跳过空格和回车**，直接读下一个非空白字符。

### 5\. cin 与 cout 方向对比

| 对象 | 运算符 | 方向 | 记忆 |
| --- | --- | --- | --- |
| cout | `<<` | 内容流向屏幕（向左） | "送出去" |
| cin | `>>` | 数据流向变量（向右） | "读进来" |

口诀：**cout 用 `<<` 往外送，cin 用 `>>` 往里读。**

## 十、用 cout 输出变量

### 1\. 输出变量的值

变量名不加引号，cout 输出的是变量里的值，不是变量名：

```cpp
int a = 10;
cout << a;        // 输出: 10（输出 a 的值，不是字母 a）

```

### 2\. 变量与字符串混合输出

和第一课一样，用 `<<` 串起来：

```cpp
int a = 10, b = 20;
cout << "a = " << a << ", b = " << b << endl;
// 输出: a = 10, b = 20

```

### 3\. 输出参与算式的变量

变量可以直接参与运算后输出：

```cpp
int a = 3, b = 4;
cout << "a + b = " << a + b << endl;   // 输出: a + b = 7
cout << "a * b = " << a * b << endl;   // 输出: a * b = 12

```

### 4\. 完整示例：读入两数输出和

```cpp
#include <iostream>
using namespace std;
int main(){
    int a, b;
    cin >> a >> b;
    cout << a + b << endl;
    return 0;
}

```

输入 `3 4`，输出 `7`。这就是洛谷 P1001（A+B Problem）的核心代码。

## 十一、算术运算

### 1\. 五个基本算术运算符

| 运算符 | 含义 | 示例 | 结果 | 适用类型 |
| --- | --- | --- | --- | --- |
| `+` | 加 | `3 + 4` | `7` | 整数、小数 |
| `-` | 减 | `10 - 3` | `7` | 整数、小数 |
| `*` | 乘 | `6 * 7` | `42` | 整数、小数 |
| `/` | 除 | `15 / 4` | `3`（见下） | 整数、小数 |
| `%` | 取余（取模） | `15 % 4` | `3` | **仅整数** |

### 2\. 整数除法（重点坑点）

**两个整数相除，结果还是整数，小数部分直接丢弃（不是四舍五入）。**

```cpp
cout << 15 / 4;      // 输出 3，不是 3.75
cout << 7 / 2;       // 输出 3，不是 3.5
cout << 1 / 2;       // 输出 0

```

**怎么得到小数？** 只要有一个是小数（浮点），就按小数算：

```cpp
cout << 15.0 / 4;    // 输出 3.75
cout << 7 / 2.0;     // 输出 3.5
cout << 1.0 / 2;     // 输出 0.5

```

> **规律：** 整数 ÷ 整数 = 整数（丢小数）；只要有一边是小数 = 小数。变量同理：`int a=7; cout << a/2;` 输出 3；`double x=7; cout << x/2;` 输出 3.5。

### 3\. 取余运算 `%`

`%` 求的是除法的**余数**。`15 % 4` 读作"**15 取余 4**"，或"**15 除以 4 的余数**"（两种读法都对，课堂可任选一种统一使用）。

```cpp
cout << 15 % 4;      // 15÷4=3余1，输出 1
cout << 7 % 2;       // 7÷2=3余1，输出 1
cout << 10 % 5;      // 10÷5=2余0，输出 0

```

> **重要：** `%` 只能用于整数，不能用于小数。`3.14 % 2` 会编译错误。

**求余的常见用途：**

-   判断奇偶：`n % 2 == 0` 是偶数，`n % 2 == 1` 是奇数。
-   取个位：`n % 10` 得到 n 的个位数字（如 `123 % 10 = 3`）。
-   取十位：`n / 10 % 10`（如 `123 / 10 = 12`，`12 % 10 = 2`）。

### 4\. 运算优先级

和数学一样：**先乘除求余，后加减**，有括号先算括号。

```cpp
cout << 2 + 3 * 4;       // 输出 14，不是 20（先算 3*4=12，再 2+12）
cout << (2 + 3) * 4;     // 输出 20（括号改变顺序）
cout << 10 - 2 * 3;      // 输出 4（先 2*3=6，再 10-6）

```

> **建议：** 拿不准优先级时，**加括号**，既不会错，也更好读。

### 5\. 字符与整数的运算（拓展）

char 本质存的是 ASCII 码，所以可以和整数运算：

```cpp
char ch = 'a';           // 'a' 的 ASCII 码是 97
cout << char(ch - 32);   // 97-32=65，即 'A'，输出 A（小写转大写）

```

小写转大写的规律：小写字母比对应大写字母大 32。`'a'-'A'` = 32。这是洛谷 P5704（字母转换）的原理。

### 6\. 复合赋值与自增（拓展，信奥常用）

| 写法 | 等价于 | 含义 |
| --- | --- | --- |
| `a += 5` | `a = a + 5` | 加 5 后赋回 |
| `a -= 2` | `a = a - 2` | 减 2 后赋回 |
| `a *= 3` | `a = a * 3` | 乘 3 后赋回 |
| `a++` | `a = a + 1` | 自增 1 |
| `a--` | `a = a - 1` | 自减 1 |

```cpp
int a = 10;
a += 5;        // a 变成 15
a++;           // a 变成 16

```

## 十二、常见错误与注意事项

| 编号 | 错误 | 后果 | 改正 |
| --- | --- | --- | --- |
| E1 | 变量未定义就使用 | 编译错误 was not declared | 先 `int a;` 再用 |
| E2 | 变量未赋值就读取 | 输出垃圾值（随机数） | 定义即初始化 |
| E3 | 变量名数字开头 | 编译错误 | 改成字母/下划线开头 |
| E4 | 变量名用关键字 | 编译错误 | 换个名字 |
| E5 | 同名变量重复定义 | 编译错误 redefinition | 一个名字只定义一次 |
| E6 | char 用双引号 `"A"` | 类型不匹配（变成字符串） | 字符用单引号 `'A'` |
| E7 | int 用单引号 `'10'` | 类型错误 | 数字不加引号 |
| E8 | 整数除法丢小数 | `7/2` 得 3 而非 3.5 | 改用 `7.0/2` 或 double |
| E9 | double 用 `%` | 编译错误 | 求余只用整数 |
| E10 | cin 用 `<<` / cout 用 `>>` | 方向反，编译错误 | cin 用 `>>`，cout 用 `<<` |
| E11 | 大小写写错（`Int`） | 编译错误 | 关键字全小写 `int` |

### 输入法与标点提醒

-   写代码一律英文输入法。变量名里的中文标点、空格都会报错。
-   `=` 是赋值，`==` 才是"判断相等"（后续课程会学，先记住别混）。

## 十三、方法、套路与策略小结

### 1\. 顺序结构三步套路

信奥里最基础的题目都是"读入→计算→输出"三步：

-   **第1步 读入：** 先按数据类型定义变量，再用 `cin >>` 把键盘数据读进变量。
-   **第2步 计算：** 用算术运算把读入的数据算出结果（可存入新变量，也可直接输出）。
-   **第3步 输出：** 用 `cout <<` 把结果输出。

### 2\. 类型选择决策

-   题目说"整数"→ `int`；数据或结果可能超过 21 亿 → `long long`
-   题目说"小数/浮点/保留几位"→ `double`
-   题目说"一个字符/字母"→ `char`
-   题目说"是否/判断"→ `bool`

### 3\. 整数除法防坑三问

写除法前先问自己：

1.  两个操作数都是整数吗？
2.  题目要整数结果还是小数结果？
3.  要小数就加 `.0` 或用 double 变量。

### 4\. 信奥策略意识

-   **定义即初始化**：避免垃圾值，减少调试时间。
-   **变量名见名知意**：长题目里 `sum`、`cnt`、`ans` 比单字母好维护。
-   **拿不准加括号**：优先级记不清就加括号，绝不靠猜。
-   **样例验证**：提交前用样例输入跑一遍，逐字符比对输出。
