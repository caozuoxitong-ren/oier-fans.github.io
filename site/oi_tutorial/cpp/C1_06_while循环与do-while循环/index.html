    三、教学重点与难点

    1. 教学重点

    • while 循环语法：while (条件) { 循环体 }——条件为真时重复执行循环体。
    • 执行流程：先判断后执行——先判断条件，为真才执行循环体，为假直接退出（可能一次都不执行）。
    • while 三要素：初始化（循环外）、条件（while 括号）、更新（循环体内）——三要素分散在三处，比 for 更容易出错。
    • for 与 while 的等价转换：for (init; cond; update) { body } 等价于 init; while (cond) { body; update; }。
    • do-while 语法（拓展）：do { 循环体 } while (条件);——先执行后判断，至少执行一次。

    2. 教学难点

    • while 比 for 更容易死循环——因为更新语句不在 while 那一行，容易忘记写。
    • 理解"先判断后执行"——条件一开始就为假时，循环体一次都不执行。
    • 循环变量必须在循环外初始化——for 可以在括号内声明，while 不行。
    • do-while 的 while 后面有分号——do { ... } while (条件); 末尾的分号不能漏。
    • 求位数时 n=0 的特殊处理——while 循环对 n=0 输出 0（一次都不执行），但 0 的位数应该是 1。

    四、为什么需要 while

    1. for 循环的局限

    第 5 课学的 for 循环，三要素（起点、终点、步长）都写在一行，非常适合已知循环次数的场景：

    // 已知循环 n 次，用 for 最直观
for (int i = 1; i <= n; i++) {
    cout << i << '\n';
}

    但有些场景循环次数未知——你不知道要重复多少次，只知道"满足某个条件就继续，不满足就停"：

    • 一根木棍每天锯一半，第几天长度变为 1？——不知道要锯几天，只知道"长度大于 1 就继续锯"。
    • 一个正整数不断除以 10，除几次变成 0？——不知道要除几次，只知道"大于 0 就继续除"。
    • 角谷猜想：一个数不断变换，几步后变成 1？——不知道要变换几步，只知道"不等于 1 就继续"。

    这些场景用 for 循环写不了（因为不知道终点），需要 while 循环。

    2. while 解决"未知次数"的重复

    while 循环就是"只要条件成立，就一直重复执行"。

    // 只要 a 大于 1，就继续除以 2
while (a > 1) {
    a /= 2;
    day++;
}

    课程衔接： 三个循环中，for（第 5 课）适合已知次数，while（本课）适合未知次数，do-while（本课拓展）适合至少执行一次。下一课学循环读入数据处理（cin 在循环内），后续还有专门课次讲解 for 与分支组合、for 循环嵌套。

    五、while 循环语法

    1. 语法结构

    while (条件) {
    循环体;
}

    while 后面的括号里只有一个条件（没有初始化和更新）：

    部分 作用 写在哪里
    条件 每轮循环前判断，为真则执行循环体，为假则退出 while 括号内
    循环体 每轮执行的内容，必须包含改变循环变量的语句 花括号内

    关键区别： for 的三要素集中在一行，while 只有条件在一行——初始化在循环外，更新在循环体内。所以 while 比 for 更容易忘记更新，导致死循环。

    2. 最小示例

    // 打印 1 到 5（用 while 实现）
int i = 1;           // ① 初始化（在循环外）
while (i <= 5) {     // ② 条件（在 while 括号内）
    cout << i << '\n';
    i++;             // ③ 更新（在循环体内）
}

    输出：

    1
2
3
4
5

    理解要点： 这段代码和 for (int i = 1; i <= 5; i++) 完全等价。while 把 for 的三要素拆成了三处。

    六、while 循环执行流程

    1. 执行顺序

    while 循环的执行流程是先判断后执行：

            ↓
② 判断条件：检查条件是否为真
        ↓
   为真 → ③ 执行循环体（含更新语句）
        ↓
   回到 ② 判断条件
        ↓
   为假 → 退出循环，执行 while 后面的语句

    关键： while 是先判断后执行。如果条件一开始就为假，循环体一次都不执行。

    2. 执行次数追踪

    以 int i = 1; while (i <= 5) { cout << i; i++; } 为例：

    轮次 i 的值 判断 i\<=5 循环体执行 更新后 i
    第 1 轮 1 真 打印 1 2
    第 2 轮 2 真 打印 2 3
    第 3 轮 3 真 打印 3 4
    第 4 轮 4 真 打印 4 5
    第 5 轮 5 真 打印 5 6
    第 6 轮 6 假 不执行 退出循环

    和 for 循环的追踪表完全一样。while 和 for 本质上是同一种循环，只是写法不同。

    3. 条件一开始就为假

    int i = 10;
while (i <= 5) {       // 10 <= 5 为假
    cout << i << '\n'; // 一次都不执行
    i++;
}
cout << "结束" << '\n'; // 直接执行这里

    输出：结束（循环体一次都没执行）。

    while 的特点： 条件一开始就为假时，循环体执行 0 次。这是 while 和 do-while 的核心区别（do-while 至少执行 1 次）。

    七、while 循环三要素

    while 循环也有三要素，和 for 一样，但分散在三处：

    要素 for 中的位置 while 中的位置 容易出错吗
    初始化 for 括号内 int i = 1 循环外单独一行 容易忘记
    条件 for 括号内 i <= n while 括号内 i <= n 不容易忘
    更新 for 括号内 i++ 循环体内某一行 最容易忘记

    三要素缺一不可

    // 错误：忘记初始化，i 的值不确定
while (i <= 5) {     // i 没赋初值，行为不确定
    cout << i;
    i++;
}

// 错误：忘记更新，死循环
int i = 1;
while (i <= 5) {     // i 永远是 1，1 <= 5 永远为真
    cout << i;       // 无限打印 1
}

// 正确：三要素齐全
int i = 1;           // ① 初始化
while (i <= 5) {     // ② 条件
    cout << i;
    i++;             // ③ 更新
}

    死循环第一杀手： while 循环忘记写更新语句，循环变量永远不变，条件永远为真，程序卡死。写完 while 立刻检查循环体内有没有改变循环变量的语句。

    八、for 与 while 的对比与转换

    1. 等价转换

    任何 for 循环都可以改写成 while 循环：

    // for 写法
for (int i = 1; i <= n; i++) {
    cout << i << '\n';
}

// 等价的 while 写法
int i = 1;              // 初始化移到循环外
while (i <= n) {        // 条件不变
    cout << i << '\n';
    i++;                // 更新移到循环体末尾
}

    转换口诀： 初始化移到外面，条件照抄，更新移到循环体最后。

    2. 什么时候用 for，什么时候用 while

    特性 for 循环 while 循环
    三要素位置 集中在一行 分散在三处
    适合场景 已知循环次数 未知循环次数
    典型例子 打印 1 到 n、求 1 到 n 的和 一尺之棰、求位数、角谷猜想
    忘记更新的风险 低（更新在 for 行） 高（更新在循环体内）
    条件一开始为假 循环体执行 0 次 循环体执行 0 次

    选择建议： 已知循环次数用 for（更简洁、更不容易出错），未知循环次数用 while（更自然）。能用 for 就用 for，for 写不了再用 while。

    九、基本应用 1：求正整数 n 的位数

    题目

    读入一个正整数 n（n ≥ 0），求 n 的位数。如 n=12345 输出 5，n=0 输出 1。

    分析

    不断除以 10，每除一次位数减一（去掉末位），直到 n 变成 0。除的次数就是位数。

    但 n=0 时，while (n > 0) 条件一开始就为假，循环体执行 0 次，输出 0——这是错的，0 的位数是 1。所以需要特殊处理。

    代码

    #include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int count = 0;
    if (n == 0) {
        count = 1;         // 0 的位数是 1，特殊处理
    } else {
        while (n > 0) {
            n /= 10;       // 每次去掉末位
            count++;       // 位数加 1
        }
    }
    cout << count << '\n';
    return 0;
}

    输入输出示例：

    输入: 12345
输出: 5

输入: 0
输出: 1

    while 的特点暴露： n=0 时循环体执行 0 次，输出 0，这是 while "先判断后执行"的副作用。后面学 do-while 可以自然解决（至少执行一次）。

    十、基本应用 2：一尺之棰

    题目

    《庄子》："一尺之棰，日取其半，万世不竭。" 第一天有一根长度为 a 的木棍，从第二天开始每天锯掉一半（除以 2，向下取整）。第几天木棍长度变为 1？

    分析

    不知道要锯几天，只知道"长度大于 1 就继续锯"——典型的 while 场景。

    代码

    #include <iostream>
using namespace std;
int main() {
    int a;
    cin >> a;
    int day = 1;            // 第一天长度为 a
    while (a > 1) {         // 长度大于 1 就继续
        a /= 2;             // 锯一半（向下取整）
        day++;              // 过了一天
    }
    cout << day << '\n';
    return 0;
}

    输入输出示例：

    输入: 100
输出: 7

    洛谷题号： P5720 [深基4.例4] 一尺之棰。入口：https://www.luogu.com.cn/problem/P5720。

    为什么用 while 不用 for： 不知道要锯几天（循环次数未知），只知道"长度大于 1 就继续"。如果用 for，终点写不出来。

    十一、基本应用 3：翻转数字

    题目

    读入一个正整数 n，将其各位数字翻转输出。如 n=12345 输出 54321，n=1200 输出 21（前导 0 不保留）。

    分析

    不断取末位（n % 10），拼到结果变量的末尾（rev = rev * 10 + n % 10），然后去掉末位（n /= 10）。重复直到 n 变成 0。

    代码

    #include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int rev = 0;            // 翻转结果，累加器初始化为 0
    while (n > 0) {
        rev = rev * 10 + n % 10;  // 取末位拼到 rev 后面
        n /= 10;                  // 去掉末位
    }
    cout << rev << '\n';
    return 0;
}

    输入输出示例：

    输入: 12345
输出: 54321

输入: 1200
输出: 21

    执行过程追踪（n=123）

    轮次 n n%10 rev 变化 n/=10 后
    开始 123 — 0 —
    第 1 轮 123 3 0×10+3=3 12
    第 2 轮 12 2 3×10+2=32 1
    第 3 轮 1 1 32×10+1=321 0
    第 4 轮 0 — 退出循环 —

    累加器模式的变体： rev = rev * 10 + n % 10 是累加器的变体——每次把 rev 乘以 10（左移一位）再加上新取的末位。和第 5 课的 sum += i 本质相同，都是"用循环逐步累积结果"。

    十二、do-while 循环（拓展）

    1. 语法结构

    do {
    循环体;
} while (条件);    // 注意末尾有分号！

    do-while 先执行循环体，再判断条件。条件为真则继续，为假则退出。

    部分 作用 写在哪里
    循环体 先执行一次，之后每轮判断条件为真才执行 do 花括号内
    条件 循环体执行完后判断，为真则继续，为假则退出 while 括号内

    关键： do-while 是先执行后判断，循环体至少执行一次。这是 do-while 和 while 的核心区别。

    2. 执行顺序

    ① 执行循环体（含更新语句）
        ↓
② 判断条件
        ↓
   为真 → 回到 ①
        ↓
   为假 → 退出循环

    3. 最小示例

    // 打印 1 到 5（用 do-while 实现）
int i = 1;
do {
    cout << i << '\n';
    i++;
} while (i <= 5);    // 注意末尾分号！

    4. do-while 解决求位数问题

    do-while 至少执行一次，可以自然处理 n=0 的情况：

    // 用 do-while 求位数，n=0 也能正确输出 1
int n;
cin >> n;
int count = 0;
do {
    n /= 10;
    count++;
} while (n > 0);      // 至少执行一次，n=0 时 count=1
cout << count << '\n';

    do-while 的价值： 当"至少执行一次"时，do-while 比 while 更自然——不需要特殊处理 n=0。但信奥中 do-while 用得较少，while 更常见。

    十三、while 与 do-while 的对比

    特性 while do-while
    执行顺序 先判断后执行 先执行后判断
    至少执行一次 否（条件一开始为假时执行 0 次） 是（至少执行 1 次）
    末尾分号 不需要 需要（} while (条件);）
    适合场景 可能不需要执行 至少需要执行一次
    典型例子 一尺之棰、翻转数字 求位数（含 n=0）、菜单选择
    信奥中使用频率 高 低

    选择建议： 大多数场景用 while。只有当"至少执行一次"时才考虑 do-while。信奥中 do-while 主要用于求位数、输入验证等场景。

    十四、常见错误与注意事项

    编号 错误 后果 改正
    E1 忘记在循环体内写更新语句 循环变量不变，死循环 循环体内必须有改变循环变量的语句
    E2 循环变量未初始化 值不确定，行为不可预测 while 前必须初始化循环变量
    E3 条件写成赋值 while (i = 1) = 是赋值，结果非 0 即真，死循环 条件用 ==、<、> 等，不能用 =
    E4 while 后多加分号 循环体为空，只判断条件不执行语句 删掉 while 行末的分号
    E5 do-while 的 while 后忘记分号 编译错误 } while (条件); 末尾必须有分号
    E6 循环体只有一条语句但没加花括号 更新语句不在循环体内，死循环 一律加花括号
    E7 条件永远为真 死循环 确保循环体内有改变条件的语句
    E8 求位数时 n=0 未特殊处理 输出 0，正确答案是 1 用 if 特殊处理，或用 do-while
    E9 while 条件写反（> 写成 <） 循环不执行或死循环 仔细检查条件方向
    E10 累加器/累乘器在 while 外初始化 每次循环重置，结果错误 累加器在循环外初始化一次

    死循环

    while 循环最常见的错误就是死循环——条件永远为真，循环永远不结束：

    // 死循环：忘记 i++，i 永远是 1，1 <= 5 永远为真
int i = 1;
while (i <= 5) {
    cout << i;
}

// 死循环：条件写成赋值，i=1 结果非 0 即真
int i = 1;
while (i = 1) {     // 这是赋值不是判断！
    cout << i;
    i++;
}

    调试建议： 程序卡住不动，几乎一定是死循环。检查 while 循环体内是否有改变循环变量的语句，条件是否会变为假。

    while 后多分号

    // 错误：while 后多了分号，循环体为空
int i = 1;
while (i <= 5); {      // 分号使循环体为空！
    cout << i;         // 这不在循环体内！
    i++;
}

    while 后面多了 ;，{} 变成了普通代码块而非循环体。循环只执行空语句（什么都不做），且 i 不变，导致死循环。

    十五、方法、套路与策略小结

    1. while 循环三步套路

    • 第 1 步 确定条件： 什么情况下继续循环？什么情况下退出？
    • 第 2 步 写循环体： 每轮要做什么（计算、更新循环变量）。
    • 第 3 步 检查三要素： 初始化（循环外）、条件（while 括号）、更新（循环体内）——三要素是否齐全。

    2. for 与 while 选择决策

    场景 选择 理由
    已知循环次数（打印 1 到 n、求和） for 三要素集中，更简洁
    未知循环次数（一尺之棰、求位数） while 终点未知，for 写不了
    至少执行一次（求位数含 n=0） do-while 先执行后判断
    可能不执行（条件一开始可能为假） while 先判断后执行

    3. while 经典套路

    套路 模式 典型题目
    不断缩小直到满足条件 while (条件) { n 缩小; } 一尺之棰（n/=2）、角谷猜想
    不断取末位 while (n > 0) { 取 n%10; n/=10; } 求位数、翻转数字
    累加器 while (条件) { sum += ...; } 用 while 求和
    计数器 while (条件) { count++; ...; } 统计循环执行次数

    4. 信奥策略意识

    • 三要素检查： 写完 while 立刻检查——初始化在循环外、条件在括号内、更新在循环体内。
    • 死循环排查： 程序卡住不动，先查 while 循环体内有没有改变循环变量。
    • 边界必测： 测 n=0（最小值）和 n=较大值，检查循环是否正确退出。
    • for 优先： 能用 for 就用 for（更不容易死循环），for 写不了再用 while。
    • 性能意识： 循环内频繁输出用 '\n' 而非 endl。
    • 样例验证： 手动追踪循环变量和累加器的变化，确认循环执行次数和输出结果。

    附录 · 本课代码模板速查

    模板 1：while 循环万能框架

    #include <iostream>
using namespace std;
int main() {
    // 初始化循环变量
    // while (条件) {
    //     循环体;
    //     更新循环变量;    // 不能忘！
    // }
    return 0;
}

    三要素：初始化（循环外）、条件（while 括号）、更新（循环体内）。写完立刻检查三要素是否齐全。

    模板 2：for 改写 while

    // for 写法
for (int i = 1; i <= n; i++) {
    // 循环体
}

// 等价 while 写法
int i = 1;              // 初始化移到外面
while (i <= n) {        // 条件不变
    // 循环体
    i++;                // 更新移到循环体末尾
}

    转换口诀：初始化移到外面，条件照抄，更新移到循环体最后。

    模板 3：不断缩小直到满足条件

    // 一尺之棰模式：不断缩小 n 直到满足退出条件
while (n > 1) {         // 条件：n 大于 1 就继续
    n /= 2;             // 缩小 n
    count++;            // 统计次数
}

    适合"不知道要重复几次，只知道满足条件就停"的场景。

    模板 4：不断取末位

    // 求位数 / 翻转数字模式：不断取末位直到 n 变成 0
while (n > 0) {
    int digit = n % 10;  // 取末位
    // 用 digit 做事（统计、拼接等）
    n /= 10;             // 去掉末位
}

    适合"逐位处理数字"的场景。注意 n=0 时循环体执行 0 次。

    模板 5：do-while 万能框架

    // do-while：至少执行一次
do {
    // 循环体;
    // 更新循环变量;
} while (条件);    // 注意末尾分号！

    do-while 先执行后判断，至少执行一次。适合"不管条件如何，先做一轮再说"的场景。

    模板 6：do-while 求位数（含 n=0）

    int n;
cin >> n;
int count = 0;
do {
    n /= 10;
    count++;
} while (n > 0);      // 至少执行一次，n=0 时 count=1
cout << count << '\n';

    do-while 自然处理 n=0，不需要 if 特殊处理。这是 do-while 相对 while 的优势场景。
