## 三、教学重点与难点

### 1\. 教学重点

-   数组声明三要素：类型、数组名、大小（必须是编译期常量）。
-   下标从 0 开始，有效范围 `0` 到 `大小 - 1`，越界访问是数组第一杀手。
-   用 for 循环完成数组读入与输出，循环变量同时充当下标。
-   数组遍历求和：累加器初始化为 0，遍历 `a[0]` 到 `a[n-1]`。
-   逆序输出：倒序 for 循环，从 `n - 1` 递减到 `0`。

### 2\. 教学难点

-   下标 0 起始与"第几个"的对应关系——第 1 个元素是 `a[0]`，第 n 个元素是 `a[n-1]`。
-   部分初始化规则：未列出的元素自动补 0，而非随机值。
-   数组大小必须用常量，不能用 `cin >> n` 读入的变量——`int a[n];` 是非标准写法。
-   `sizeof(a)` 返回数组总字节数而非元素个数，求元素个数要用 `sizeof(a) / sizeof(a[0])`。
-   逆序循环的边界：从 `n - 1` 开始（不是 `n`），到 `0` 结束（条件是 `i >= 0`，不是 `i > 0`）。

## 四、分模块展开

### 模块1　为什么需要数组

回顾第 7 课：我们学会了用循环读入多组数据。但那些数据读入后"用完即丢"——比如求和时，每个数读进来就立刻加到累加器里，加完就不再需要了。

但很多题目要求"先存下来，之后再处理"。例如：

-   读入 n 个数，**倒着输出**——必须先把所有数存住，才能从最后一个开始往前输出。
-   读入 n 个数，**先求和再求平均值，最后统计大于平均值的个数**——第一遍求和用完，第二遍还要逐个比较，数据不能丢。
-   斐波那契数列：把每一项存进数组，之后任意查询第几项都能直接回答。

如果只靠变量，存 100 个数就要定义 100 个变量——`int a1, a2, a3, ... a100;`，这显然不现实。**数组**就是一次性定义一组同类型变量的工具：一个名字管一排数据，用下标区分每一个。

> **从第 7 课到第 8 课的跨越：** 第 7 课解决"怎么读入多组数据"，第 8 课解决"怎么把多组数据存下来反复用"。

### 模块2　一维数组的声明

**语法：**

```cpp
类型 数组名[大小];

```

**最小示例：**

```cpp
int a[100];       // 定义一个能存 100 个 int 的数组
double b[50];     // 定义一个能存 50 个 double 的数组
long long c[105]; // 定义一个能存 105 个 long long 的数组

```

**要点：**

-   **类型**：数组中每个元素的类型，`int`、`double`、`long long`、`char` 均可。
-   **数组名**：标识符，命名规则与变量相同，见名知意。
-   **大小**：数组能容纳的元素个数，**必须是编译期常量**——要么是字面量（如 `100`），要么是 `const` 常量。不能是运行时读入的变量。

**推荐写法（用 const 常量定义大小）：**

```cpp
const int MAXN = 1005;
int a[MAXN];     // 标准写法，大小是 const 常量

```

> **坑点：** `int n; cin >> n; int a[n];` 这种"大小用变量"的写法叫变长数组（VLA），不是标准 C++，只是 GCC 编译器的扩展功能。信奥评测环境通常能编译，但属于不规范写法，考试中可能踩坑。正确做法：开一个比最大需求稍大的常量大小数组，实际只用前 n 个位置。

### 模块3　数组下标与 0 起始约定

数组中的每个元素用**下标**（索引）来访问。

**语法：**

```cpp
数组名[下标]

```

**核心规则：下标从 0 开始。**

对于大小为 `N` 的数组 `a`，有效下标范围是 `0` 到 `N - 1`：

| 下标 | `a[0]` | `a[1]` | `a[2]` | ... | `a[N-1]` |
| --- | --- | --- | --- | --- | --- |
| 含义 | 第 1 个 | 第 2 个 | 第 3 个 | ... | 第 N 个 |

**最小示例：**

```cpp
int a[5];
a[0] = 10;    // 给第 1 个元素赋值
a[1] = 20;    // 给第 2 个元素赋值
a[4] = 50;    // 给第 5 个（最后一个）元素赋值
cout << a[0]; // 输出第 1 个元素：10

```

> **第一杀手——越界访问：** `a[5]` 对于大小为 5 的数组是越界的（有效下标只有 0~4）。C++ 不会自动检查越界，程序不会报错，但会访问到非法内存，导致输出乱码、程序崩溃甚至不可预测的行为。**牢记：大小为 N，最大下标是 N-1。**

> **对应关系易错：** "第 1 个元素"是 `a[0]`，"第 n 个元素"是 `a[n-1]`。读题时"第几个"要减 1 才是下标。

### 模块4　数组的初始化

**完整初始化：**

```cpp
int a[5] = {1, 2, 3, 4, 5};   // a[0]=1, a[1]=2, a[2]=3, a[3]=4, a[4]=5

```

**部分初始化（重点）：**

```cpp
int a[5] = {1, 2};   // a[0]=1, a[1]=2, a[2]=0, a[3]=0, a[4]=0

```

> **规则：** 花括号内列出的值依次赋给前面的元素，**未列出的元素自动初始化为 0**，而非随机值。这是部分初始化最重要的性质。

**全部清零的惯用写法：**

```cpp
int a[100] = {0};    // a[0] 显式赋 0，其余元素自动补 0 → 全部为 0
int b[100] = {};     // 同样全部清零（C++ 标准支持）

```

> **不初始化的后果：** `int a[100];`（不带 `= {...}`）时，数组元素的值是未定义的（垃圾值），直接读取会得到不可预测的结果。**养成习惯：数组声明时一律初始化，至少写 `= {0}`。**

### 模块5　数组的循环读入

用 for 循环读入 n 个数到数组，循环变量同时充当下标。

**标准写法：**

```cpp
int n;
cin >> n;                 // 先读入元素个数
for (int i = 0; i < n; i++) {
    cin >> a[i];          // 依次读入到 a[0], a[1], ..., a[n-1]
}

```

**完整程序：**

```cpp
#include <iostream>
using namespace std;
const int MAXN = 105;
int a[MAXN];
int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];
    for (int i = 0; i < n; i++) cout << a[i] << " ";
    return 0;
}

```

输入:

```
5
10 20 30 40 50

```

输出:

```
10 20 30 40 50

```

> **要点：** 循环条件是 `i < n`（不是 `i <= n`），因为下标范围是 0 到 n-1。这与第 5 课的边界意识一脉相承——`<` 还是 `<=` 要想清楚。

---

### 模块6　数组的循环输出

**正序输出：**

```cpp
for (int i = 0; i < n; i++) {
    cout << a[i];
    if (i < n - 1) cout << " ";   // 元素之间空格，末尾不加多余空格
}
cout << '\n';

```

> **输出格式意识：** 很多题目要求元素之间用空格分隔、行末不能有多余空格。上面用 `if (i < n - 1)` 控制空格。如果题目不严格要求行末空格，简单写 `cout << a[i] << " ";` 也可以。循环内输出用 `'\n'` 而非 `endl`，保持第 5 课建立的性能意识。

### 模块7　数组遍历：求和与求最值

**求和（无分支，本课核心）：**

```cpp
int sum = 0;                          // 累加器初始化为 0（第 5 课铁律）
for (int i = 0; i < n; i++) {
    sum += a[i];                      // 把每个元素累加到 sum
}
cout << sum << '\n';

```

**求最大值（用到第 3 课基本 if）：**

```cpp
int maxv = a[0];                      // 用第一个元素做初始最大值
for (int i = 1; i < n; i++) {         // 从第二个元素开始比较
    if (a[i] > maxv) maxv = a[i];     // 发现更大的就更新
}
cout << maxv << '\n';

```

> **说明：** 求最大值用到了第 3 课已学的基本 if。for 循环与分支语句的系统性组合（包括 continue、break）是第 9 课的内容，本课只做初步应用。求最小值同理：把 `a[0]` 做初始值，条件改成 `a[i] < minv`。

> **初始值技巧：** 求最值时，初始值用 `a[0]` 比用"一个很小的数"更稳妥——不需要猜数据范围，不会因为猜错而出错。循环从 `i = 1` 开始（第 0 个已经用作初始值）。

### 模块8　数组逆序输出

逆序输出是数组最经典的应用——只有把数据存进数组，才能"从后往前"访问。

**倒序 for 循环：**

```cpp
for (int i = n - 1; i >= 0; i--) {
    cout << a[i] << " ";
}
cout << '\n';

```

**完整程序：**

```cpp
#include <iostream>
using namespace std;
const int MAXN = 105;
int a[MAXN];
int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) cin >> a[i];     // 正序读入
    for (int i = n - 1; i >= 0; i--) cout << a[i] << " ";  // 倒序输出
    return 0;
}

```

输入:

```
5
8 6 5 4 1

```

输出:

```
1 4 5 6 8

```

> **边界要点：** 倒序循环从 `i = n - 1` 开始（最后一个元素的下标），到 `i = 0` 结束（第一个元素的下标），条件是 `i >= 0`。初学者常写成 `i = n`（越界）或 `i > 0`（漏掉 `a[0]`）。

> **为什么第 7 课做不到？** 第 7 课的循环读入是"读一个处理一个"，数据用完即丢。逆序输出要求"先存住全部，再从后往前输出"——必须用数组。这正是数组的不可替代之处。

## 五、常见错误与注意事项

| 编号 | 错误 | 后果 | 改正 |
| --- | --- | --- | --- |
| 1 | `int a[5]; a[5] = 10;` | 越界访问，程序崩溃或乱码 | 有效下标是 0~4，最大下标是大小减 1 |
| 2 | `int n; cin >> n; int a[n];` | 变长数组，非标准 C++ | 用 `const int MAXN = 1005; int a[MAXN];` |
| 3 | `int a[100];` 不初始化 | 元素是垃圾值，读取结果不可预测 | 声明时写 `= {0}` 清零 |
| 4 | `for (int i = 0; i <= n; i++) cin >> a[i];` | `a[n]` 越界 | 条件改为 `i < n` |
| 5 | 逆序循环 `for (int i = n; i >= 0; i--)` | `a[n]` 越界 | 从 `i = n - 1` 开始 |
| 6 | 逆序循环 `for (int i = n - 1; i > 0; i--)` | 漏掉 `a[0]` | 条件改为 `i >= 0` |
| 7 | `int a[5] = {1, 2, 3, 4, 5, 6};` | 初始值个数超过大小，编译警告/报错 | 初始值个数不能超过数组大小 |
| 8 | 以为 `sizeof(a)` 是元素个数 | 得到总字节数（如 400 而非 100） | 元素个数 = `sizeof(a) / sizeof(a[0])` |
| 9 | 求和时累加器忘记初始化 | sum 是垃圾值，结果错误 | `int sum = 0;`（第 5 课铁律） |
| 10 | 求最值时初始值设为 0 | 全负数组会得到错误最大值 | 用 `a[0]` 做初始值，从 `i = 1` 开始 |

> **补充说明——sizeof 的正确用法：** 对于 `int a[100]`，`sizeof(a)` 的值是 `100 * sizeof(int) = 400`（字节），不是元素个数 100。求元素个数的标准写法是 `sizeof(a) / sizeof(a[0])`。不过在信奥中，我们通常直接用 `n`（实际读入的元素个数）控制循环，很少用 sizeof 求数组长度。

## 六、方法、套路与策略小结

### 1\. 数组题三步走

数组题的标准结构，与第 7 课的"读入—计算—输出"三步一脉相承，但数据要先存入数组：

| 步骤 | 操作 | 代码骨架 |
| --- | --- | --- |
| 第 1 步 | 读入 n 与数组 | `cin >> n; for (int i = 0; i < n; i++) cin >> a[i];` |
| 第 2 步 | 遍历数组计算 | `for (int i = 0; i < n; i++) { ... }` |
| 第 3 步 | 输出结果 | `cout << 结果;` |

### 2\. 下标边界决策表

| 场景 | 起始值 | 条件 | 说明 |
| --- | --- | --- | --- |
| 正序遍历 | `i = 0` | `i < n` | 访问 a\[0\] 到 a\[n-1\] |
| 倒序输出 | `i = n - 1` | `i >= 0` | 从最后一个到第一个 |
| 求最值 | `i = 1` | `i < n` | a\[0\] 已用作初始值 |

### 3\. 策略意识

-   **性能意识：** 循环内输出用 `'\n'` 而非 `endl`；数组开在全局区（大数组开全局避免栈溢出）。
-   **规范意识：** 数组大小用 `const` 常量定义；声明时一律初始化 `= {0}`；下标从 0 开始，时刻检查边界。
-   **策略意识：** 先存后用——数据需要反复使用时存入数组；求最值用 `a[0]` 做初始值，不猜数据范围。


**自编3 · 求最大值**

题目：输入第一行为正整数 n（1 ≤ n ≤ 100），第二行为 n 个整数。输出这 n 个整数中的最大值。

输入:

```
5

```

输出:

```

```

**自编4 · 大于平均值的个数**

题目：输入第一行为正整数 n（1 ≤ n ≤ 100），第二行为 n 个整数（均为 int 范围）。先求出这 n 个整数的平均值（实数），再统计并输出大于平均值的整数个数。

输入:

```
5
1 2 3 4 5

```

输出:

```
2

```

> **说明：** 自编4 需要两遍遍历数组——第一遍求和算平均值，第二遍逐个比较统计个数。数据存入数组后可以反复使用，这正是数组的价值。平均值是实数，比较时注意 int 与 double 的隐式转换。
