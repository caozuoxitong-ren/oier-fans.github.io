    第 1 课 · 程序设计入门

    程序框架 · 基本输出 · 混合输出 · 图形打印 · 分级题单

    一、教学目标

    本课目标依据布鲁姆教育目标分类学（修订版）制定，采用 ABCD 撰写法（主体 A、行为动词 B、行为条件 C、表现程度 D），确保目标可观测、可评估。

    1. 课时目标（ABCD 表述）

    编号 目标表述（ABCD） 分类定位
    G1 记忆 在给出 C++ 程序框架的条件下，学生能够准确写出 #include、using namespace std、int main、return 0 四个要素，无遗漏。 事实性知识 × 记忆
    G2 理解 在给出 cout 语句的条件下，学生能够解释 <<（插入运算符）的作用与数据流动方向，并用"把右边的内容送进输出流"表述。 概念性知识 × 理解
    G3 应用 在无参考的条件下，学生能够独立编写 cout 语句输出数字、字符串、算式结果、多内容链式输出与换行，语法正确率不低于 90%。 程序性知识 × 应用
    G4 应用 在给出图形样例的条件下，学生能够用多行 cout 拼出正方形、三角形等基本字符图形，输出与样例逐行一致。 程序性知识 × 应用
    G5 分析 在给出含错误代码的条件下，学生能够区分中文标点、缺分号、缺引号、引号不配对等典型错误并改正。 程序性知识 × 分析

    2. 目标说明

    • 认知层次： 本课为第一课，认知重心在"记忆—理解—应用"三层，分析层以纠错形式渗透，不设评价/创造层目标。
    • 知识类型： 重点训练程序性知识（写代码的技能），概念性知识（理解 << 与输出流）作为支撑。
    • 边界： 学完本课即可完成纯输出类题目；输入（cin）留待下一课，题单 B 级题作为衔接预告。

    二、教学内容概览

    • 模块1　C++ 程序基本框架（四要素 + 逐行解释）
    • 模块2　cout 基本语法与 << 运算符

    • 模块4　输出多个内容（链式 <<）
    • 模块5　输出换行（endl 与 \n 的区别与选择）
    • 模块6　字符串与数字混合输出
    • 模块7　输出基本图形（正方形、三角形）
    • 模块8　常见错误与注意事项
    • 模块9　方法、套路与策略小结
    • 模块10　分级分类练习题单（洛谷 + 自编）

    三、教学重点与难点

    1. 教学重点

    • C++ 程序框架四要素：#include <iostream>、using namespace std;、int main(){...}、return 0;。
    • cout 语句的基本形式：cout << 内容;，以及 << 的链式使用。
    • 字符串用双引号、字符用单引号、数字不加引号三者的区别。
    • 换行的两种写法 endl 与 \n，以及信奥中优先用 \n 的性能意识。

    2. 教学难点

    • 理解 << 是"把右边送进左边输出流"的方向感，初学者易与数学"小于号"混淆。
    • 中英文标点混用导致编译错误（中文分号、中文引号、中文括号），是最常见的"玄学报错"。
    • 混合输出时字符串与数字的拼接顺序与 << 的对应关系。
    • 图形打印中"逐行输出、每行末尾换行"的拆解思路。

    四、C++ 程序基本框架

    任何一个 C++ 程序都套在下面这个"骨架"里。先记住骨架，再往 main 函数里填内容。

    1. 通用代码框架（背下来）

    #include <iostream>         // 1. 包含头文件
using namespace std;        // 2. 使用标准命名空间
int main(){                 // 3. 主函数，程序从这里开始
    // 在这里写你的代码

    return 0;               // 4. 正常结束，返回 0
}

    2. 逐行解释

    行 代码 作用
    1 #include <iostream> 引入输入输出库，cout、cin、endl 都在里面。没有这行，cout 用不了。
    2 using namespace std; 声明使用标准命名空间，这样写 cout 就行；否则要写 std::cout。
    3 int main(){ 主函数，程序执行的入口，所有代码从这里开始跑。
    4 // 你的代码 花括号 { } 内部是你写的语句，每条语句以分号 ; 结尾。
    5 return 0; 告诉系统"程序正常跑完了"，返回 0。

    3. 框架记忆口诀

    "一引库、二命名、三主函数、四返回。" ——四句话，缺一不可。

    4. 最小程序：输出一句话

    把框架填满，就是一个能跑的程序：

    #include <iostream>
using namespace std;
int main(){
    cout << "Hello, World!";
    return 0;
}

    运行结果（屏幕显示）：Hello, World!

    五、cout 输出语句

    1. 基本语法

    cout 是"标准输出流"对象，<< 叫"插入运算符"（也叫流插入符）。

    读法： 把 << 右边的内容，送进左边的 cout（屏幕）。

    cout << 要输出的内容;

    要点：

    • 每条 cout 语句以分号 ; 结尾（C++ 每条语句都要分号）。
    • << 的方向是"向左"，形象理解：内容流向屏幕。
    • cout 与 << 之间可以有空格，习惯写 cout <<。

    2. 输出一个数

    数字不加引号，直接写。

    cout << 42;       // 输出: 42
cout << 2024;     // 输出: 2024
cout << 3.14;     // 输出: 3.14（小数）

    注意： 42 是数字，能参与运算；"42" 是字符串（见下），只是长得像数字，不能运算。

    3. 输出一个字符串

    字符串用英文双引号 " " 括起来。

    cout << "Hello";          // 输出: Hello
cout << "信息学奥赛";      // 输出: 信息学奥赛
cout << "3+4";            // 输出: 3+4（原样输出，不计算）

    关键区别： 双引号里的内容原样照印，"3+4" 输出的是字符 3、+、4，不是 7。

    4. 输出一个算式的结果

    数字和运算符不加引号，cout 会先算再输出。

    cout << 3 + 4;      // 输出: 7（先算 3+4=7，再输出 7）
cout << 10 - 3;     // 输出: 7
cout << 6 * 7;      // 输出: 42
cout << 15 / 4;     // 输出: 3（整数除法，只取商，不四舍五入）
cout << 15 % 4;     // 输出: 3（% 是求余数，15÷4=3余3）

    易错点： 15/4 在 C++ 中整数除整数得整数（3），不是 3.75。要得小数需写成 15.0/4。

    5. 输出多个内容（链式 <<）

    一条 cout 可以用多个 << 串起来，依次输出。

    cout << "a" << "b" << "c";      // 输出: abc
cout << 1 << 2 << 3;            // 输出: 123（不是 6，是拼接显示）
cout << "数学" << 3 + 4;        // 输出: 数学7

    理解： 每个 << 把右边那一项送进 cout，多项之间默认没有空格，紧挨着输出。

    6. 输出换行

    两种写法都能换行：

    cout << "第一行" << endl;      // endl 换行
cout << "第二行" << "\n";      // \n 换行（写在字符串里）
cout << "第三行\n";            // 也可以直接写在字符串末尾
    写法 含义 特点
    endl 换到下一行 会刷新输出缓冲区，速度略慢
    "\n" 换行符（字符串内） 只换行不刷新，速度更快
    '\n' 换行符（单字符） 与 "\n" 等效，单引号表示单个字符

    信奥建议： 竞赛中输出量大时优先用 "\n" 或 '\n'，比 endl 快，避免因输出慢导致 TLE（超时）。平时练习两种都可。

    7. 字符串与数字混合输出

    这是最常用的写法：用 << 把字符串、算式、数字拼成一句话。

    cout << "3 + 4 = " << 3 + 4 << endl;
// 输出: 3 + 4 = 7

    拆解：

    • "3 + 4 = " 是字符串，原样照印（注意 = 两边有空格，空格也是字符串的一部分）。
    • 3 + 4 是算式，先算得 7，再输出 7。
    • endl 最后换行。

    更多例子：

    cout << "小明今年 " << 12 << " 岁。" << "\n";
// 输出: 小明今年 12 岁。

cout << "长 = " << 5 << "，宽 = " << 3 << "，面积 = " << 5 * 3 << "\n";
// 输出: 长 = 5，宽 = 3，面积 = 15

    六、输出基本图形

    图形打印的核心思路：把图形拆成"一行一行"，每行用一个 cout 输出，行末换行。

    1. 正方形（3×3）

    目标图形：

    ***
***
***

    代码：每行一个 cout，行末 endl。

    #include <iostream>
using namespace std;
int main(){
    cout << "***" << endl;
    cout << "***" << endl;
    cout << "***" << endl;
    return 0;
}

    2. 直角三角形（左对齐）

    目标图形：

    *
**
***
****

    代码：

    #include <iostream>
using namespace std;
int main(){
    cout << "*"  << endl;
    cout << "**" << endl;
    cout << "***" << endl;
    cout << "****" << endl;
    return 0;
}

    3. 等腰三角形（居中）

    目标图形（4 行）：

       *
  ***
 *****
*******

    思路：每行 = 若干空格 + 若干星号。空格用字符串里的空格表示即可。

    #include <iostream>
using namespace std;
int main(){
    cout << "   *"    << endl;   // 3空格 + 1星
    cout << "  ***"   << endl;   // 2空格 + 3星
    cout << " *****"  << endl;   // 1空格 + 5星
    cout << "*******" << endl;   // 0空格 + 7星
    return 0;
}

    技巧： 数清楚每行"空格数 + 星号数"，对齐就靠空格。星号每行多 2 个，空格每行少 1 个。

    4. 打印矩形边框（空心正方形）

    目标图形（4×4 边框）：

    ****
*  *
*  *
****


    #include <iostream>
using namespace std;
int main(){
    cout << "****"  << endl;
    cout << "*  *"  << endl;   // 中间是 星+2空格+星
    cout << "*  *"  << endl;
    cout << "****"  << endl;
    return 0;
}

    七、常见错误与注意事项

    以下是初学者最常踩的坑，全部会导致编译错误或输出错误，务必逐条记住。

    编号 错误 后果 改正
    E1 中文分号 ； 编译错误，报一堆看不懂的错 用英文分号 ;
    E2 中文引号 “ ” 字符串识别失败，编译错误 用英文双引号 " "
    E3 漏写分号 ; 编译错误 expected ; 每条语句末尾加 ;
    E4 引号不配对 编译错误 missing terminating 成对写引号，先打一对再填内容
    E5 漏写 #include <iostream> cout 未声明 框架第一行不能少
    E6 漏写 using namespace std; cout 不在作用域 框架第二行不能少
    E7 把 << 写成 >> 方向反，编译错误 << 是输出，>> 是输入
    E8 数字加引号 "3+4" 输出 3+4 而非 7 要计算就不加引号
    E9 字符串加引号但想计算 原样照印 算式放引号外
    E10 main 写成 mian 链接错误，找不到入口 拼写仔细，main

    输入法提醒

    写代码时一律用英文输入法（半角）。 中文标点（；“” （））和英文标点（; "" ()）长得像但完全不同，是新手第一杀手。

    八、方法、套路与策略小结

    1. 写 cout 的三步套路

    • 定内容： 第1步，想清楚要输出什么（一句话 / 一个数 / 一个图形）。
    • 分类型： 第2步，分清哪些是"原样照印"的字符串（加引号），哪些是"要计算"的算式（不加引号）。
    • 串与收尾： 第3步，用 << 串起来，末尾加 endl 或 \n 换行，句末加分号。

    2. 图形打印套路

    • 把图形按行拆，每行一个 cout。
    • 每行 = 前导空格 + 图案字符，数清楚数量。
    • 行末一定要换行，否则全挤在一行。
    • 复杂图形先在纸上画出来，标好每行空格数和字符数，再翻译成代码。

    3. 信奥策略意识（从第一课开始培养）

    • 性能优化： 换行优先用 \n 而非 endl，养成性能意识。
    • 样例验证： 提交前对照样例逐字符检查，包括空格、大小写、标点。
    • 格式注意： 程序末尾换行可有可无，但建议有；多余的空格会导致格式错误。
    • 排错技巧： 看到一堆报错先查第一行，多半是中文标点或漏分号。
