## 四、什么是数位拆分——从具体问题引入

### 从一个具体问题说起

给定整数 123，怎么求它的各位数字之和（1 + 2 + 3 = 6）？

如果数字只有 3 位，可以直接拆：

```cpp
int bai = 123 / 100;       // 百位 = 1
int shi = 123 / 10 % 10;   // 十位 = 2
int ge  = 123 % 10;         // 个位 = 3
int sum = bai + shi + ge;   // 和 = 6

```

但如果数字有 10 位、20 位呢？一位一位地写除法显然不现实。我们需要一种**通用的、不管几位数都能用**的方法——这就是**数位拆分**。

### 数位拆分的定义

**数位拆分**是指把一个整数的各位数字分别提取出来的操作。它是信奥入门的经典专题，很多题目都涉及：

-   求各位数字之和
-   数字逆序（123 → 321）
-   回文数判断（121 正读反读都一样）
-   统计某数字出现次数
-   统计一个数有几位

> **核心思想**：一个十进制整数，它的最后一位可以用 `% 10` 取出，去掉最后一位可以用 `/ 10` 实现。反复执行这两个操作，就能逐位提取所有数字。

## 五、方法一：取余整除法（while 循环法）

### 核心口诀

> **取余得个位，整除去个位。**

-   `n % 10`：取出 n 的最后一位（个位），结果范围 0~9。
-   `n /= 10`：去掉 n 的最后一位（等价于 `n = n / 10`）。
-   用 while 循环反复执行，直到 n 变为 0。

### 基本模板

```cpp
int n;
cin >> n;
while (n > 0) {
    int digit = n % 10;  // 取最后一位
    cout << digit << ' ';
    n /= 10;             // 去掉最后一位
}

```

**输入：**

```
123

```

**输出：**

```
3 2 1

```

### 执行过程详解

以 n = 123 为例：

| 轮次 | n 的值 | n % 10（取出） | n /= 10（剩余） | 输出 |
| --- | --- | --- | --- | --- |
| 1 | 123 | 3 | 12 | `3` |
| 2 | 12 | 2 | 1 | `2` |
| 3 | 1 | 1 | 0 | `1` |

3 轮后 n = 0，while 条件 `n > 0` 为假，循环结束。

> **注意**：while 取余法是**从低位到高位**提取的（先取个位，再取十位，再取百位……）。输出顺序与原数字的位数顺序相反。

### 应用：各位数字之和

把取出的每一位累加到 sum 中：

```cpp
#include <iostream>
using namespace std;
int main() {
    int n, sum = 0;
    cin >> n;
    while (n > 0) {
        sum += n % 10;  // 累加各位
        n /= 10;
    }
    cout << sum << endl;
    return 0;
}

```

**输入：** `123`

**输出：** `6`

> **技巧**：累加器 `sum` 必须初始化为 0。这是第 5 课累加器思想在数位拆分中的应用。

### 0 的特殊处理

当 n = 0 时，`while (n > 0)` 的条件一开始就为假，循环体一次都不执行。这意味着：

-   0 的各位数字之和 = 0（sum 保持初始值 0）→ **正确**
-   0 的数字逆序 = 0（ans 保持初始值 0）→ **正确**
-   0 的位数 = 0（count 保持初始值 0）→ **错误！0 应该是 1 位数**

所以统计位数时需要特判 0：

```cpp
int count = 0;
if (n == 0) count = 1;  // 特判：0 是 1 位数
while (n > 0) {
    count++;
    n /= 10;
}

```

> **易错点**：0 是唯一一个 `while (n > 0)` 不进入循环的非空输入。涉及"位数"时必须特判 0。

## 六、方法二：字符串法

### 基本思路

把数字当作字符串读入，每个字符就是一个数字位。用 `s[i] - '0'` 把字符转成数字。

```cpp
#include <iostream>
#include <string>
using namespace std;
int main() {
    string s;
    cin >> s;
    int sum = 0;
    int len = s.length();
    for (int i = 0; i < len; i++) {
        sum += s[i] - '0';  // 字符转数字
    }
    cout << sum << endl;
    return 0;
}

```

**输入：** `123`

**输出：** `6`

### 字符转数字的原理

字符在内存中存的是 ASCII 码。`'0'` 的 ASCII 码是 48，`'1'` 是 49，`'2'` 是 50，以此类推。

| 字符 | ASCII 码 | 减 '0' 后 |
| --- | --- | --- |
| `'0'` | 48 | 0 |
| `'1'` | 49 | 1 |
| `'2'` | 50 | 2 |
| ... | ... | ... |
| `'9'` | 57 | 9 |

所以 `s[i] - '0'` 就把字符变成了对应的数字。

> **易错点**：直接用 `s[i]` 参与运算得到的是 ASCII 码（48~57），不是数字本身（0~9）。必须减 `'0'`。

### 字符串法的优缺点

| 优点 | 缺点 |
| --- | --- |
| 从高位到低位顺序处理 | 不能直接做算术运算（如逆序拼接） |
| 代码简单直观 | 需要额外引入 `string` 类型 |
| 可以处理前导零 | 比较和排序需要额外转换 |

> **适用场景**：需要从高位到低位顺序处理、不需要对数字做算术运算时，字符串法更简单。

## 七、方法三：固定位数法

### 基本思路

当数字的位数已知且固定时，可以直接用除法和取余按位提取，不需要循环。

**3 位数为例：**

```cpp
int n;
cin >> n;  // 假设是 3 位数
int bai = n / 100;       // 百位
int shi = n / 10 % 10;   // 十位
int ge  = n % 10;         // 个位
cout << bai << ' ' << shi << ' ' << ge << endl;

```

**输入：** `123`

**输出：** `1 2 3`

### 提取规则

| 目标位 | 公式 | 示例（n=123） |
| --- | --- | --- |
| 个位 | `n % 10` | 3 |
| 十位 | `n / 10 % 10` | 2 |
| 百位 | `n / 100` | 1 |
| 千位 | `n / 1000` | — |
| 第 k 位（从右起） | `n / 10^(k-1) % 10` | — |

> **技巧**：从右起第 k 位的通用公式是"先除以 10 的 k−1 次方，再对 10 取余"。个位 k=1（除以 1 再取余），十位 k=2（除以 10 再取余），百位 k=3（除以 100 再取余）。

> **局限**：固定位数法只适用于位数已知且固定的场景。如果数字位数不确定，应该用 while 取余法。

## 八、三种方法对比与选择

### 对比表

| 方法 | 处理顺序 | 需要循环 | 适用场景 | 优点 | 缺点 |
| --- | --- | --- | --- | --- | --- |
| while 取余法 | 低位→高位 | 是（while） | 通用，任意位数 | 最通用，可做算术 | 逆序提取 |
| 字符串法 | 高位→低位 | 是（for） | 需要顺序处理 | 简单直观 | 不能直接算术 |
| 固定位数法 | 按需选取 | 否 | 位数固定 | 代码最简 | 不通用 |

### 方法选择决策表

| 场景 | 推荐方法 | 理由 |
| --- | --- | --- |
| 求各位数字之和 | while 取余法 | 不需要顺序，通用 |
| 数字逆序 | while 取余法 | 逆序提取正好用于拼接 |
| 回文数判断 | while 取余法 | 逆序后与原数比较 |
| 统计某数字出现次数 | while 取余法 | 通用，逐位判断 |
| 位数统计 | while 取余法 | 通用（注意特判 0） |
| 提取特定数位（如百位） | 固定位数法 | 直接计算，最简 |
| 位数固定时的各位之和 | 固定位数法 | 代码最简 |
| 需要从高位到低位顺序处理 | 字符串法 | 天然顺序 |

> **策略意识**：绝大多数数位拆分题用 while 取余法就能解决。字符串法和固定位数法是特定场景的优化选择。拿不准时，先用 while 取余法。

## 九、经典应用——数字逆序与回文数判断

### 数字逆序

把 123 变成 321，核心是**逆序拼接公式**：

> **ans = ans \* 10 + digit**

原理：每次取出一位，把它"接"到结果的末尾。乘以 10 把已有数字左移一位，加上新数字填入个位。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n, ans = 0;
    cin >> n;
    while (n > 0) {
        ans = ans * 10 + n % 10;  // 逆序拼接
        n /= 10;
    }
    cout << ans << endl;
    return 0;
}

```

**输入：** `123`

**输出：** `321`

### 执行过程详解

以 n = 123 为例：

| 轮次 | n | n % 10（digit） | ans = ans\*10 + digit | n /= 10 |
| --- | --- | --- | --- | --- |
| 1 | 123 | 3 | 0×10+3 = **3** | 12 |
| 2 | 12 | 2 | 3×10+2 = **32** | 1 |
| 3 | 1 | 1 | 32×10+1 = **321** | 0 |

3 轮后 ans = 321，n = 0，循环结束。

> **关键理解**：`ans * 10` 把 ans 已有的数字整体左移一位（个位变十位、十位变百位……），`+ digit` 把新取出的数字填入空出来的个位。这和"取出"操作正好是互逆的。

> **前导零自动去除**：输入 100，逆序过程为 0→0→1，最终 ans = 1。前导零自然消失，因为 0×10+0 = 0。这符合整数逆序的常见要求。

### 洛谷 P1307 数字反转

洛谷 P1307 \[NOIP2011 普及组\] 数字反转就是数字逆序的经典题。需要注意两点：

1.  **负数保留符号**：先记录符号，对绝对值做逆序，最后加回符号。
2.  **前导零自动去除**：逆序拼接公式天然处理，无需额外操作。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int sign = 1;
    if (n < 0) { sign = -1; n = -n; }  // 记录符号，取绝对值
    int ans = 0;
    while (n > 0) {
        ans = ans * 10 + n % 10;
        n /= 10;
    }
    cout << ans * sign << endl;  // 加回符号
    return 0;
}

```

**输入：** `-380`

**输出：** `-83`

### 回文数判断

**回文数**是指正读和反读都一样的数，如 121、1331、7。

判断策略：**逆序后与原数比较**。如果逆序数等于原数，就是回文数。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int original = n, reversed = 0;  // 保存原数，逆序用
    while (n > 0) {
        reversed = reversed * 10 + n % 10;
        n /= 10;
    }
    if (reversed == original) cout << "Yes" << endl;
    else cout << "No" << endl;
    return 0;
}

```

**输入：** `121`

**输出：** `Yes`

> **技巧**：必须用 `original` 保存原数，因为 while 循环会修改 n。逆序结束后拿 `reversed` 和 `original` 比较，而不是和已经被修改的 n 比较。

> **注意**：回文数判断还有一种"只比较一半"的优化方法（只反转后半部分，与前半部分比较），但入门阶段用"全部逆序后比较"更直观、更不容易出错。

## 十、经典应用——数字统计与位数统计

### 统计某数字出现次数

统计数字 x 在 n 中出现了多少次：逐位取余，每次判断是否等于 x。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n, x;
    cin >> n >> x;
    int count = 0;
    while (n > 0) {
        if (n % 10 == x) count++;  // 当前位等于x就计数
        n /= 10;
    }
    cout << count << endl;
    return 0;
}

```

**输入：** `11223 2`

**输出：** `2`

### 位数统计

统计一个数有几位：每次 `/ 10` 去掉一位，计数器加 1。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int count = 0;
    if (n == 0) count = 1;  // 特判 0
    while (n > 0) {
        count++;
        n /= 10;
    }
    cout << count << endl;
    return 0;
}

```

**输入：** `12345`

**输出：** `5`

> **易错点**：0 是 1 位数，但 `while (n > 0)` 不进入循环，count 保持 0。必须用 `if (n == 0) count = 1` 特判。

## 十一、综合应用——多组数据的数位处理

### 洛谷 P1980 计数问题

洛谷 P1980 \[NOIP2013 普及组\] 计数问题：计算 1 到 n 的所有整数中，数字 x 共出现了多少次。

这道题综合了**第 10 课的循环嵌套**和**本课的数位拆分**：

-   外层 for 循环：遍历 1 到 n 的每个数。
-   内层 while 循环：对每个数做数位拆分，逐位判断是否等于 x。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n, x;
    cin >> n >> x;
    int count = 0;
    for (int i = 1; i <= n; i++) {
        int t = i;           // 用临时变量 t，不破坏 i
        while (t > 0) {
            if (t % 10 == x) count++;
            t /= 10;
        }
    }
    cout << count << endl;
    return 0;
}

```

**输入：** `11 1`

**输出：** `4`

> **技巧**：内层循环用临时变量 `t = i` 而非直接用 `i`，因为 while 会修改变量（`t /= 10`）。如果直接用 `i`，外层 for 循环的计数器就被破坏了。这是"保护循环变量"的重要习惯。

### 洛谷 B4036 数位之和

洛谷 B4036 \[GESP202409 二级\] 数位之和：给定 n 个正整数，判断每个数的各位数字之和是否为 7 的倍数。

```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    for (int i = 0; i < n; i++) {
        int num;
        cin >> num;
        int sum = 0;
        int t = num;          // 临时变量
        while (t > 0) {
            sum += t % 10;
            t /= 10;
        }
        if (sum % 7 == 0) cout << "Yes" << endl;
        else cout << "No" << endl;
    }
    return 0;
}

```

> **策略意识**：多组数据的数位处理，套路固定：外层循环读入每个数 → 用临时变量做数位拆分 → 累加/计数/判断 → 输出结果。临时变量保护原值是关键。

## 十二、常见错误与注意事项

| 编号 | 错误 | 后果 | 改正 |
| --- | --- | --- | --- |
| 1 | 忘记 `n /= 10` | n 永远不变，死循环 | while 体内必须有 `n /= 10` |
| 2 | while 条件写成 `n > 1` | 漏掉个位为 1 的情况 | 条件应为 `n > 0` |
| 3 | 0 的位数统计错误 | 0 被统计为 0 位 | `if (n == 0) count = 1` 特判 |
| 4 | 字符串法忘记 `s[i] - '0'` | 得到 ASCII 码 48~57 | 必须减 `'0'` 转数字 |
| 5 | 逆序拼接用加法 `ans + digit` | 结果完全错误 | 用 `ans = ans * 10 + digit` |
| 6 | `% 10` 结果范围记错 | 误以为 1~10 | 实际是 0~9 |
| 7 | 累加器/计数器未初始化 | 随机值 | `sum = 0`、`count = 0` |
| 8 | 回文数比较用修改后的 n | 永远不相等 | 用 `original` 保存原数 |
| 9 | 内层 while 直接用外层 for 的 i | i 被破坏，外层循环混乱 | 用临时变量 `t = i` |
| 10 | 固定位数法十位写成 `n / 10` | 得到的是去掉个位后的数 | 十位应为 `n / 10 % 10` |

> **重点展开——逆序拼接公式**：`ans = ans * 10 + digit` 是本课最重要的公式。常见错误是写成 `ans + digit`（只加不乘，没有移位）或 `ans * 10`（只乘不加，没有填入新数字）。记住：**乘 10 移位 + 加数填个位**，两步缺一不可。

> **重点展开——0 的特殊性**：0 是唯一让 `while (n > 0)` 不进入循环的自然数。对于求和、逆序，0 的结果是 0（正确）；对于位数统计，0 的结果是 0（错误，应为 1）。涉及"位数"时必须特判。

## 十三、方法、套路与策略小结

### 结构套路

**数位拆分三步走**（while 取余法万能框架）：

1.  取位：`digit = n % 10`
2.  处理：累加 / 计数 / 拼接 / 判断
3.  去位：`n /= 10`

**逆序拼接公式**：`ans = ans * 10 + digit`（乘 10 移位 + 加数填个位）。

**回文数判断套路**：保存原数 → 逆序 → 比较。

**多组数据套路**：外层循环读入 → 临时变量拆分 → 处理 → 输出。

### 方法选择决策表

| 场景 | 推荐方法 | 关键操作 |
| --- | --- | --- |
| 各位数字之和 | while 取余法 | `sum += n % 10` |
| 数字逆序 | while 取余法 | `ans = ans * 10 + n % 10` |
| 回文数判断 | while 取余法 | 逆序后与原数比较 |
| 数字出现次数 | while 取余法 | `if (n % 10 == x) count++` |
| 位数统计 | while 取余法 | `count++`（特判 0） |
| 提取特定数位 | 固定位数法 | `n / 10^k % 10` |
| 顺序处理各位 | 字符串法 | `s[i] - '0'` |

### 信奥策略意识

-   **性能意识**：while 取余法的循环次数等于数字位数，效率很高（10^9 的数只有 10 位）。
-   **规范意识**：累加器初始化为 0；临时变量保护循环变量；`n /= 10` 不要漏写。
-   **策略意识**：拿不准方法时先用 while 取余法；涉及位数时特判 0；逆序拼接公式要理解原理而非死记。
