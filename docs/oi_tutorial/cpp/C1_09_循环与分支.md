## 三、教学重点与难点

### 1\. 教学重点

-   循环 + if 的基本组合：在循环体内用 if 做条件判断，满足条件执行特定操作。
-   continue 的语义：跳过本次循环剩余语句，进入下一次迭代。
-   break 的语义：直接结束当前整个循环，执行循环后的语句。
-   条件求和与条件计数的标准写法：累加器/计数器 + if 判断。
-   求最值的标准写法：用 `a[0]` 做初始值，遍历比较更新。
-   查找搜索的标准写法：标记变量记录结果，找到用 break 提前退出。

### 2\. 教学难点

-   continue 在 for 与 while 中的差异：for 中 continue 后仍执行 `i++`；while 中 continue 后直接回到条件判断，若更新语句在 continue 之后会被跳过，导致死循环。
-   break 只跳出最内层循环（本课单层循环暂不涉及嵌套，但需建立意识）。
-   break 与 continue 的混淆：break 是"不循环了"，continue 是"这一次不做了，下一次继续"。
-   求最值时初始值的选择：用 `a[0]` 而非 0 或随意猜测的值。
-   查找题中标记变量的使用：用 `ans = -1` 或 `bool found` 记录是否找到，循环后据此决定输出。

## 四、分模块展开

### 模块1　为什么需要循环 + 分支

回顾前几课：

-   第 5~6 课学了 for、while、do-while 三种循环，但循环体内主要做累加、累乘等**无分支操作**。
-   第 7 课的循环读入，循环体也只是读入和累加，没有 if 判断。
-   第 8 课的数组遍历，求和时无需 if；但求最大值时已经用到了第 3 课的基本 if——`if (a[i] > maxv) maxv = a[i];`。

实际问题中，循环体内经常需要**根据条件做不同处理**：

-   统计 1~100 中偶数有多少个——要判断每个数是否为偶数。
-   求 m 到 n 之间奇数之和——只累加奇数，跳过偶数。
-   在一组数中找到第一个大于 10 的数——找到就停，不必继续。

这些都需要**循环 + 分支**的组合。本课正式、系统地讲解这一组合，并引入两个控制语句 continue 和 break。

> **从第 8 课到第 9 课的跨越：** 第 8 课的求最值已经用了循环 + if，但只是初步应用；第 9 课系统化讲解组合模式，并引入 continue/break 两个控制语句，覆盖一批经典题型。

### 模块2　循环与分支的基本组合模式

**基本模式：在循环体内用 if 做条件判断。**

```cpp
for (int i = 开始; i < 结束; i++) {
    if (条件) {
        // 满足条件时执行的操作
    }
}

```

**最小示例——统计 1~10 中偶数个数：**

```cpp
#include <iostream>
using namespace std;
int main() {
    int cnt = 0;                          // 计数器初始化为 0
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0) {                 // 判断是否为偶数
            cnt++;                        // 满足条件则计数
        }
    }
    cout << cnt << '\n';                  // 输出 5
    return 0;
}

```

> **要点：** 这就是循环 + 分支最基本的组合——循环负责遍历，if 负责筛选。累加器/计数器在循环外初始化（第 5 课铁律），if 在循环体内做条件判断。`i % 2 == 0` 判断偶数，`i % 2 == 1` 判断奇数（注意用 `==` 不是 `=`）。

### 模块3　continue：跳过本次

**语法：**

```cpp
continue;

```

**语义：跳过本次循环剩余语句，直接进入下一次迭代。**

-   对于 for 循环：continue 后**仍执行更新表达式**（如 `i++`），再判断条件。
-   对于 while 循环：continue 后**直接回到条件判断**，若更新语句写在 continue 之后会被跳过。

**示例——求 1~10 中奇数之和（用 continue 跳过偶数）：**

```cpp
#include <iostream>
using namespace std;
int main() {
    int sum = 0;
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0) continue;         // 偶数则跳过，i++ 仍执行
        sum += i;                         // 只有奇数会执行到这里
    }
    cout << sum << '\n';                  // 1+3+5+7+9 = 25
    return 0;
}

```

> **for 中 continue 的执行流程：** 遇到 `continue` 时，跳过本次循环体剩余语句（`sum += i`），但 for 头部的 `i++` 仍然执行，然后判断 `i <= 10`。所以不会死循环。

> **while 中 continue 的陷阱：** 在 while 循环中，如果循环变量的更新写在 continue 之后，continue 会跳过更新，导致死循环。详见模块8常见错误。

> **continue 的本质：** "这一次不做了，下一次继续。"它不结束循环，只跳过当前这一轮。

### 模块4　break：提前结束循环

**语法：**

```cpp
break;

```

**语义：直接结束当前整个循环，执行循环之后的语句。**

**示例——在一组数中找到第一个大于 10 的数：**

```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int found = -1;                       // 标记是否找到，-1 表示未找到
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        if (x > 10) {                     // 找到第一个大于 10 的
            found = x;                    // 记录下来
            break;                        // 立即结束循环，不再读后面的数
        }
    }
    if (found != -1) cout << found << '\n';
    else cout << "未找到" << '\n';
    return 0;
}

```

输入:

```
5
3 7 15 2 8

```

输出:

```
15

```

> **break 的本质：** "不循环了，退出。"与 continue 的"这一次不做了"不同，break 是彻底结束循环。找到答案后无需继续搜索时，用 break 提前退出，既正确又高效。

> **break 与哨兵终止：** 第 7 课的哨兵终止读入 `while (cin >> x)` 中，如果用 `while (1)` 写法，就需要 `if (x == 0) break;` 来退出——这是 break 的典型应用场景。

> **break 只跳出最内层：** 如果以后学到循环嵌套，break 只结束它所在的那一层循环，外层不受影响。本课只学单层循环，暂不涉及此问题，但需建立意识。

### 模块5　经典题型一：条件求和与条件计数

**条件求和：累加满足条件的数。**

```cpp
int sum = 0;
for (int i = m; i <= n; i++) {
    if (i % 2 == 1) {                     // 条件：奇数
        sum += i;                         // 满足条件才累加
    }
}
cout << sum << '\n';

```

**条件计数：统计满足条件的个数。**

```cpp
int cnt = 0;
for (int i = 1; i <= n; i++) {
    if (i % 2 == 0) {                     // 条件：偶数
        cnt++;                            // 满足条件则计数器加 1
    }
}
cout << cnt << '\n';

```

> **套路对比：** 条件求和用 `sum += i`，条件计数用 `cnt++`。两者结构完全一样——循环遍历 + if 筛选 + 累加/计数。区别只在累加的是"值"还是"1"。

> **continue 的等价写法：** 条件求和也可以用 continue 反向写——`if (i % 2 == 0) continue; sum += i;`（跳过偶数，只累加奇数）。两种写法等价，选顺手的即可。

> **分类问题：** 若要把数据分成两类分别统计（如能被 k 整除的 A 类、不能整除的 B 类），在同一个循环里用 `if ... else` 分别累加和计数即可，不必写两个循环。

### 模块6　经典题型二：求最值

**求最大值（第 8 课已初步接触，此处系统化）：**

```cpp
int maxv = a[0];                          // 用第一个元素做初始值
for (int i = 1; i < n; i++) {             // 从第二个开始比较
    if (a[i] > maxv) {                    // 发现更大的
        maxv = a[i];                      // 更新最大值
    }
}
cout << maxv << '\n';

```

**求最小值：**

```cpp
int minv = a[0];
for (int i = 1; i < n; i++) {
    if (a[i] < minv) {                    // 发现更小的
        minv = a[i];                      // 更新最小值
    }
}
cout << minv << '\n';

```

> **初始值策略：** 用 `a[0]` 做初始值是最稳妥的写法——不需要猜数据范围，不会因为猜错而出错（比如初始值设为 0，但数据全为负数就会出错）。循环从 `i = 1` 开始，因为第 0 个已用作初始值。

> **同时求最大和最小：** 很多题目（如打分去最高最低）需要同时求最大值和最小值。只需在同一个循环里写两个 if：

```cpp
int maxv = a[0], minv = a[0], sum = 0;
for (int i = 0; i < n; i++) {
    if (a[i] > maxv) maxv = a[i];
    if (a[i] < minv) minv = a[i];
    sum += a[i];
}
// 去掉一个最高分一个最低分后的平均值
double avg = (double)(sum - maxv - minv) / (n - 2);

```

### 模块7　经典题型三：查找与搜索

**查找第一个满足条件的数，找到就 break：**

```cpp
int ans = -1;                             // -1 表示未找到
for (int i = 0; i < n; i++) {
    if (a[i] > 10) {                      // 条件
        ans = a[i];                       // 记录答案
        break;                            // 找到就停，不必继续
    }
}
if (ans != -1) cout << ans << '\n';
else cout << "未找到" << '\n';

```

> **查找 vs 遍历：** 遍历（求和、求最值）需要看完所有元素；查找只需要找到第一个满足条件的，找到就 break 退出。这是 break 最典型的应用——"找到就停"。

> **标记变量：** 查找题常用一个标记变量（如 `ans = -1` 或 `bool found = false`）记录是否找到。循环后根据标记变量决定输出结果还是"未找到"。

> **查找最后一个满足条件的数：** 若要找"最后一个"，则不能找到就 break，而要遍历到底，每次满足条件都更新答案——`if (条件) ans = a[i];`（不 break）。注意与"找第一个"的区别。

## 五、常见错误与注意事项

| 编号 | 错误 | 后果 | 改正 |
| --- | --- | --- | --- |
| 1 | while 中 continue 前忘记更新循环变量 | 死循环——continue 跳过更新语句，条件永远为真 | 更新语句写在 continue 之前，或改用 for 循环 |
| 2 | 混淆 break 与 continue | break 结束整个循环，continue 只跳过本次——用错导致逻辑错误 | 记住：break = "不循环了"，continue = "这一次不做了" |
| 3 | `if (i % 2 = 0)` | `=` 是赋值不是比较，条件恒为假（0 赋值后为假） | 用 `==`：`if (i % 2 == 0)` |
| 4 | 求最大值初始值设为 0，数据全为负 | 最大值错误地输出 0 | 用 `a[0]` 做初始值 |
| 5 | 查找题不用 break，遍历完所有元素 | 逻辑可能正确但效率低；若找"第一个"可能覆盖答案 | 找到第一个就用 break 退出 |
| 6 | break 以为能跳出多层循环 | break 只跳出最内层，外层继续 | 单层循环无此问题；嵌套循环需用标记变量配合 |

> **重点展开——while 中 continue 的死循环陷阱：**

```cpp
// 错误写法——死循环！
int i = 0;
while (i < 10) {
    i++;
    if (i == 5) continue;     // continue 后跳过下面的语句，但 i 已更新，不会死循环
    cout << i << " ";
}
// 上面其实不会死循环，因为 i++ 在 continue 之前

// 真正危险的写法——死循环！
int i = 0;
while (i < 10) {
    if (i == 5) continue;     // continue 后跳过 i++，i 永远是 5
    cout << i << " ";
    i++;                      // 这行被 continue 跳过了！
}

```

> **结论：** 在 while 循环中使用 continue 时，务必确保循环变量的更新语句不会被 continue 跳过。最安全的做法：要么把更新写在 continue 之前，要么直接用 for 循环（for 的更新在头部，continue 跳不过）。

## 六、方法、套路与策略小结

### 1\. 循环 + 分支题三步走

| 步骤 | 操作 | 代码骨架 |
| --- | --- | --- |
| 第 1 步 | 读入数据（到数组或逐个读入） | `cin >> n; for (int i = 0; i < n; i++) cin >> a[i];` |
| 第 2 步 | 循环遍历 + if 判断 + 累加/计数/更新 | `for (...) { if (条件) { ... } }` |
| 第 3 步 | 输出结果 | `cout << 结果;` |

### 2\. continue 与 break 决策表

| 需求 | 用哪个 | 说明 |
| --- | --- | --- |
| 跳过不满足条件的元素，继续循环 | continue | "这一次不做了，下一次继续" |
| 找到答案后立即结束循环 | break | "不循环了，退出" |
| 哨兵终止读入（while(1) 写法） | break | `if (x == 哨兵值) break;` |
| 统计满足条件的个数 | 不用 continue/break | 用 if + cnt++ 即可 |

### 3\. 经典题型模板对照

| 题型 | 核心结构 | 关键点 |
| --- | --- | --- |
| 条件求和 | `sum += i;`（在 if 内） | 累加器初始化为 0 |
| 条件计数 | `cnt++;`（在 if 内） | 计数器初始化为 0 |
| 求最值 | `if (a[i] > maxv) maxv = a[i];` | 初始值用 a\[0\]，从 i=1 开始 |
| 查找搜索 | `if (条件) { ans = x; break; }` | 标记变量记录是否找到 |

### 4\. 策略意识

-   **性能意识：** 查找题找到就用 break 退出，不必白跑；循环内输出用 `'\n'` 而非 `endl`（减少刷新缓冲区开销）。
-   **规范意识：** 累加器/计数器在循环外初始化；if 一律加花括号；`==` 与 `=` 严格区分。
-   **策略意识：** 求最值用 `a[0]` 做初始值不猜范围；查找题用标记变量记录结果；while 中用 continue 时确保更新不被跳过。

**自编2 · 跳过 3 的倍数求和**

题目：输入一个正整数 n（1 ≤ n ≤ 1000），求 1 到 n（含 n）中所有**不是 3 的倍数**的整数之和。要求使用 continue 语句跳过 3 的倍数。

输入:

```

```

输出:

```

```

> 说明：1+2+4+5+7+8+10 = 37（跳过了 3、6、9）。

**自编3 · 查找第一个 7 的倍数**

题目：输入第一行为正整数 n（1 ≤ n ≤ 100），第二行为 n 个正整数。找出这 n 个数中**第一个**是 7 的倍数的数并输出。如果不存在，输出 `None`。要求找到后用 break 退出循环。

输入:

```
6
3 5 14 8 21 7

```

输出:

```
14

```

> 说明：14 是第一个 7 的倍数，找到后立即 break，不检查后面的 21 和 7。

**自编4 · 正数个数与求和**

题目：输入第一行一个正整数 n（1 ≤ n ≤ 1000），第二行 n 个整数（每个整数绝对值不超过 1000）。统计其中**正数**（\> 0）的个数，并求所有正数之和，两个数用空格分隔输出。

输入:

```
6
3 -5 7 0 8 -2

```

输出:

```
3 18

```

> 说明：正数有 3、7、8 共 3 个，和为 3+7+8 = 18。0 既不是正数也不是负数，不计入。
