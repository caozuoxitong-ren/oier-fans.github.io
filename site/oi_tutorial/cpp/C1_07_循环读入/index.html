    第 7 课 · 循环读入数据处理

    指定个数读入 · 哨兵终止读入 · 读到 EOF · while(cin>>a) 原理 · 多变量读入 · 分级题单

    一、教学目标

    本课目标依据布鲁姆教育目标分类学（修订版）制定，采用 ABCD 撰写法（主体 A、行为动词 B、行为条件 C、表现程度 D），确保目标可观测、可评估。

    1. 课时目标（ABCD 表述）

    编号 目标表述（ABCD） 分类定位
    G1 记忆 在给出三种循环读入场景的条件下，学生能够准确说出"指定个数用 for、哨兵终止用 while、读到 EOF 用 while(cin>>a)"的对应关系，无遗漏。 事实性知识 × 记忆
    G2 理解 在给出 while (cin >> a) 代码的条件下，学生能够描述其原理（cin>>a 返回 cin，在条件处转 bool，读取成功为真、失败为假），并说明何时退出循环。 概念性知识 × 理解
    G3 应用 在无参考的条件下，学生能够用 for 循环编写"第一行 n，后跟 n 个数求和"的程序，语法正确率不低于 90%。 程序性知识 × 应用
    G4 应用 在给出哨兵终止或 EOF 终止题目的条件下，学生能够正确选择 while 写法（while (cin >> x && x != 0) 或 while (cin >> x)），累加器初始化与哨兵值处理正确。 程序性知识 × 应用
    G5 分析 在给出含错误的循环读入代码的条件下，学生能够区分"累加器在循环内初始化""哨兵值被计入结果""忘记在循环体内读入"等典型错误并改正。 程序性知识 × 分析

    2. 目标说明

    • 认知层次： 本课承接第 5、6 课的三种循环语法，认知重心在"理解—应用"。while (cin >> a) 的原理涉及流对象到 bool 的转换，是本课唯一的"概念性理解"难点。
    • 知识类型： 程序性知识（三种读入模式的代码套路）为主，概念性知识（EOF、流转换 bool）为辅。
    • 边界： 本课只学"如何用循环读入多组数据"，不涉及循环内分支判断（for + if 组合）、循环嵌套。循环内只做累加、累乘、计数等无分支操作；需要 if 判断的题目（如求最大值）放在 B 级，用第 3 课已学的基本 if。后续有专门课次讲解 for 与分支组合、for 循环嵌套。

    三、教学重点与难点

    1. 教学重点

    • 三种循环读入场景与对应循环的匹配：指定个数 → for，哨兵终止 → while，读到 EOF → while(cin>>a)。
    • 指定个数读入的标准写法：for (int i = 0; i < n; i++) { cin >> x; sum += x; }。
    • 哨兵终止读入的两种写法：while (cin >> x && x != 0) 与 while (cin >> x) { if (x == 0) break; ... }。
    • 读到 EOF 的标准写法：while (cin >> x) { ... }。
    • 累加器在循环外初始化（sum = 0 写在循环前面，不能写在循环体内）。

    2. 教学难点

    • 理解 while (cin >> a) 为什么能控制循环——cin>>a 返回 cin，在条件处被转成 bool。
    • 理解 EOF（文件结束）的概念——OJ 的输入来自文件，读完即结束。
    • 哨兵值不能计入结果——while (cin >> x && x != 0) 中 x==0 时条件为假，循环退出，0 不被处理。
    • 累加器初始化位置——写在循环体内会导致每轮清零，结果只剩最后一个数。
    • for 循环读 n 次时 i < n 与 i <= n 的区别——从 0 开始用 i < n，从 1 开始用 i <= n，都读 n 次。

    四、为什么需要循环读入

    1. 之前程序的局限

    第 1 到第 6 课的所有程序，输入都只有"一组数据"——读一个 n，或者读一个 a、一个 b，然后计算输出。比如求 1 到 n 的和：

    int n;
cin >> n;          // 只读一次
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
}
cout << sum << '\n';

    但实际的信奥题目，常常要读入很多组数据再处理：

    • 读入 n 个同学的成绩，求总分和平均分——要读 n 个数。
    • 读入每天的奖牌数，统计金牌总数——要读 n 天的数据。
    • 不断读入正整数，直到读入 0 为止，求这些数的和——不知道要读几个。
    • 不断读入整数直到输入结束，求和——连"读到什么时候停"都不知道。

    这些场景都需要循环读入——把 cin >> x 放在循环里，反复读取。

    2. 三种循环读入场景

    根据"要读几个数据"和"什么时候停"，循环读入分三种场景：

    场景 数据个数 用什么循环 典型题目
    指定个数 已知 n 个 for 循环 读 n 个数求和、统计奖牌
    哨兵终止 未知，读到特殊值停 while 循环 读到 0 为止求和
    读到 EOF 未知，读到文件结束 while(cin>>a) 不断读 a 求和

    课程衔接： 第 5 课学 for 循环，第 6 课学 while 和 do-while，本课把循环和读入结合起来。三种循环语法已学完，本课重点是"在循环里读数据"的套路。后续还有专门课次讲解 for 与分支组合、for 循环嵌套。

    五、三种循环读入场景总览

    在展开每个场景之前，先用一张表看清三种场景的全貌：

    场景 题目怎么说 循环写法 何时停
    指定个数 "第一行 n，接下来 n 行每行一个数" for (int i = 0; i < n; i++) { cin >> x; ... } 读满 n 个
    哨兵终止 "读到 0 为止"（0 不统计） while (cin >> x && x != 0) { ... } 读到哨兵值 0
    读到 EOF "输入若干整数"（没说几个） while (cin >> x) { ... } 读到文件结束

    选择口诀： 题目给了 n 就用 for；题目说"读到几为止"就用 while + 哨兵判断；题目只说"若干个数"没说几个就用 while(cin>>a)。

    下面逐个展开。

    六、场景一：指定个数读入（for 循环）

    1. 场景描述

    题目明确告诉你：第一行是一个整数 n，表示有 n 个数据；接下来 n 行（或一行 n 个）每行一个数据。这种"已知个数"的场景，用 for 循环最自然——因为 for 循环就是为"已知次数"设计的。

    2. 标准写法

    int n;
cin >> n;                       // 先读个数 n
int sum = 0;                    // 累加器在循环外初始化！
for (int i = 0; i < n; i++) {   // 循环 n 次
    int x;
    cin >> x;                   // 每次读一个数
    sum += x;                   // 累加
}
cout << sum << '\n';

    关键三点： ①先读 n；②累加器 sum = 0 写在循环外；③循环 n 次，每次读一个 x 并累加。

    3. 循环变量从 0 还是从 1

    读 n 个数，for 循环有两种等价写法：

    // 写法 A：i 从 0 到 n-1，循环 n 次
for (int i = 0; i < n; i++) { cin >> x; sum += x; }

// 写法 B：i 从 1 到 n，循环 n 次
for (int i = 1; i <= n; i++) { cin >> x; sum += x; }

    两种都读 n 次，结果一样。信奥中习惯用写法 A（从 0 开始，i < n），因为后面学数组时下标从 0 开始。

    不要混用： 从 0 开始就用 i < n，从 1 开始就用 i <= n。写成 for (int i = 0; i <= n; i++) 会读 n+1 次，多读一个！

    4. 完整示例：读 n 个数求和

    #include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int sum = 0;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    输入输出示例：

    输入:
5
1 2 3 4 5
输出:
15

    执行过程（n=5，数据 1 2 3 4 5）： i=0 读 1，sum=1；i=1 读 2，sum=3；i=2 读 3，sum=6；i=3 读 4，sum=10；i=4 读 5，sum=15；i=5 不满足 i\<5，退出。共读 5 次。

    5. 多个累加器：奥运奖牌计数

    如果每行有多个数据，可以在循环体内读多个变量：

    // 每天读金、银、铜牌数，统计总数
int n;
cin >> n;
int gold = 0, silver = 0, bronze = 0;
for (int i = 0; i < n; i++) {
    int g, s, b;
    cin >> g >> s >> b;       // 一行读三个
    gold += g;
    silver += s;
    bronze += b;
}
cout << gold << ' ' << silver << ' ' << bronze << ' '
     << gold + silver + bronze << '\n';

    洛谷题号： B2058 奥运奖牌计数。入口：https://www.luogu.com.cn/problem/B2058。

    七、场景二：哨兵终止读入（while 循环）

    1. 场景描述

    题目说：不断读入数据，直到读入一个特殊值（如 0 或 -1）为止，这个特殊值不参与计算。这种"未知个数、靠哨兵值停止"的场景，用 while 循环。

    "哨兵"就是一个站岗的标记——读到它就知道输入结束了。

    2. 两种写法

    写法 A：条件里同时判断（推荐）

    int sum = 0;
int x;
while (cin >> x && x != 0) {   // 读 x，且 x 不等于 0 才继续
    sum += x;
}
cout << sum << '\n';

    cin >> x && x != 0 用了短路求值（第 4 课学的）：先执行 cin >> x，如果读取失败（EOF）则整个条件为假，退出；如果读取成功，再判断 x != 0，x 等于 0 则退出，x 不等于 0 才进入循环体。这样 0 不会被加进 sum。

    写法 B：用 break 跳出

    int sum = 0;
int x;
while (cin >> x) {             // 一直读
    if (x == 0) break;         // 读到 0 就跳出
    sum += x;
}
cout << sum << '\n';

    break 表示"立即跳出当前循环"。读到 0 时执行 break，0 不会被加进 sum。

    两种写法等价： 写法 A 更简洁，信奥中更常用；写法 B 逻辑更直白，初学者更容易理解。两种都正确，哨兵值 0 都不会被计入。

    3. 完整示例：读到 0 为止求和

    #include <iostream>
using namespace std;
int main() {
    int sum = 0;
    int x;
    while (cin >> x && x != 0) {
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    输入输出示例：

    输入:
3 5 7 0
输出:
15

    执行过程（数据 3 5 7 0）： 读 3，3!=0，sum=3；读 5，5!=0，sum=8；读 7，7!=0，sum=15；读 0，0!=0 为假，退出。0 没有被加进 sum。如果误把 0 加进去，结果会变成 15，看似一样——但如果题目要求乘积或计数，哨兵值混入就会出错。养成"哨兵值不参与计算"的习惯。

    4. 哨兵值的选择

    哨兵值通常是题目中"不可能出现"的值：

    • 求正整数和，哨兵用 0（正数里不会有 0）。
    • 求正数和，哨兵用 -1（正数里不会有 -1）。

    读题要看清哨兵值： 题目说"读到 0 为止"就用 x != 0，说"读到 -1 为止"就用 x != -1。哨兵值写错，程序要么提前停、要么停不下来。

    八、场景三：读到文件结束（while(cin>>a)）

    1. 场景描述

    题目只说"输入若干个整数"，既没说几个，也没给哨兵值——这种场景要读到输入结束为止。在 OJ（在线评测系统）中，输入数据来自一个文件，文件里的数据全读完就是"文件结束"。

    EOF = End Of File（文件结束）。读到 EOF 时，cin >> a 读不到新数据，读取失败，循环就结束了。

    2. 标准写法

    int sum = 0;
int x;
while (cin >> x) {       // 不断读 x，读到 EOF 就停
    sum += x;
}
cout << sum << '\n';

    while (cin >> x) 的意思是：不断读取 x，直到读取失败（遇到 EOF）才退出循环。

    3. 完整示例：读到 EOF 求和

    #include <iostream>
using namespace std;
int main() {
    int sum = 0;
    int x;
    while (cin >> x) {
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    输入输出示例：

    输入:
1 2 3 4 5
输出:
15

    4. 本地调试如何结束输入

    在 OJ 上，输入数据来自文件，文件读完自动触发 EOF，while (cin >> x) 自然退出。但在自己电脑上用键盘调试时，程序会一直等输入，看起来像"卡住了"——其实是在等你手动结束输入。这时需要用快捷键模拟"输入结束"：

    操作系统 快捷键 操作说明
    Windows Ctrl + Z 先输入完所有数据，换行后按 Ctrl + Z，再按回车
    Linux / macOS Ctrl + D 输入完数据后，在新行开头按 Ctrl + D（无需回车）

    操作示例（Windows，求和程序）：

    键盘输入:
1 2 3 4 5
^Z            ← 按 Ctrl+Z，屏幕显示 ^Z，再按回车
程序输出:
15

    常见疑问：

    • 程序不动了？ 不是死循环，是在等你结束输入。按上面的快捷键即可。
    • Ctrl + Z 后没反应？ 必须先按回车把当前行送进去，再在新行按 Ctrl + Z + 回车。
    • Ctrl + D 要按两次？ 如果当前行还有未送入的内容，第一次 Ctrl + D 先送出当前行，第二次才真正触发 EOF。建议先回车再按。
    • OJ 上不用管这个： OJ 的输入是文件，读完自动 EOF，不需要任何快捷键。这个操作只在本地键盘调试时用。

    5. 为什么 while(cin>>x) 能控制循环

    这是本课的核心疑问：cin >> x 不是"读入"吗？为什么能当 while 的条件用？下一节专门讲。

    九、while(cin>>a) 的原理详解

    1. cin>>a 做了两件事

    cin >> a 这个表达式其实做了两件事：

    1. 读入：从输入流读取一个值，存入变量 a。
    2. 返回：返回 cin 本身（准确说是返回 cin 的引用，类型是 istream&）。

    所以 cin >> a 既能读数据，又能"返回一个东西"——它返回的是 cin。

    2. cin 在条件处会被转成 bool

    while 的括号里需要一个"真或假"的条件。当 cin 出现在这个位置时，C++ 会把它转换成 bool（真或假）：

    • 如果上一次读取成功 → 转成 真 → 继续循环。
    • 如果上一次读取失败（遇到 EOF，或读到类型不匹配的数据）→ 转成 假 → 退出循环。
    while (cin >> x)
       └──────┬──────┘
         ① 读 x，返回 cin
              ↓
         ② cin 转成 bool
              ↓
    读取成功 → 真 → 执行循环体 → 回到 ①
    读取失败 → 假 → 退出循环

    3. 什么时候读取失败

    cin >> x 在两种情况下会失败（转成假）：

    情况 说明
    遇到 EOF 输入数据全读完，没有新数据可读
    类型不匹配 x 是 int，但读到了字母等非数字内容

    信奥中绝大多数情况是第一种——读到文件结束。

    原理小结： while (cin >> x) = "读 x；读成功了就继续，读失败了（到 EOF 了）就停"。它把"读入"和"判断是否结束"合在了一句。

    4. 给好奇的同学（拓展）

    严格地说，cin 是 istream 类的对象。istream 类里定义了一个转换函数（operator bool），当 cin 出现在需要 bool 的地方（如 while 条件），就调用这个函数：流状态正常（没有 failbit、badbit）返回 true，否则返回 false。这是 C++11 起的写法；C++11 之前用的是 operator void*，原理类似。

    零基础阶段记住"读取成功为真、失败为假"就够了，类的细节后续课程再讲。

    十、多变量读入：while(cin>>a>>b)

    1. 场景描述

    有些题目每行有多个数据，且要读到 EOF。比如"每行两个整数 a 和 b，输入若干行，输出每行 a+b"。这时用 while (cin >> a >> b)。

    2. 原理

    cin >> a >> b 等价于 (cin >> a) >> b：先读 a 返回 cin，再用 cin 读 b 返回 cin。最后 cin 出现在 while 条件处，转成 bool——两个都读成功才为真，任一个失败（EOF）就为假。

    3. 完整示例：多对 a+b

    #include <iostream>
using namespace std;
int main() {
    int a, b;
    while (cin >> a >> b) {     // 每次读一对
        cout << a + b << '\n';  // 输出它们的和
    }
    return 0;
}

    输入输出示例：

    输入:
1 2
3 4
5 6
输出:
3
7
11

    执行过程： 读 1 2，输出 3；读 3 4，输出 7；读 5 6，输出 11；再读时遇到 EOF，读取失败，退出循环。每读一对就处理一对，这是"边读边处理"的模式。

    4. 指定行数的多变量读入

    如果题目说"第一行 n，接下来 n 行每行两个数"，则用 for：

    int n;
cin >> n;
for (int i = 0; i < n; i++) {
    int a, b;
    cin >> a >> b;
    cout << a + b << '\n';
}

    区分： 给了 n 用 for（已知次数），没给 n 用 while(cin>>a>>b)（读到 EOF）。

    十一、三种场景对比与选择决策

    对比项 指定个数 哨兵终止 读到 EOF
    题目是否给个数 n 给了 n 没给 没给
    题目是否给哨兵值 没有 给了（如 0、-1） 没有
    用什么循环 for while while(cin>>a)
    标准写法 for (i=0; i<n; i++) { cin>>x; ... } while (cin>>x && x!=0) { ... } while (cin>>x) { ... }
    何时停 读满 n 个 读到哨兵值 读到 EOF
    累加器初始化位置 循环外 循环外 循环外

    选择决策表

    题目怎么说 选择
    "第一行 n，接下来 n 个数" for 循环读 n 次
    "读到 0 为止"（0 不统计） while + x != 0
    "读到 -1 为止" while + x != -1
    "输入若干整数"（没说几个） while(cin>>x)
    "每行两个数，输入若干行" while(cin>>a>>b)

    决策口诀： 给个数用 for，给哨兵用 while 加判断，啥都不给用 while(cin>>a)。

    十二、常见错误与注意事项

    编号 错误 后果 改正
    E1 累加器在循环体内初始化 每轮清零，结果只剩最后一个数 sum = 0 写在循环外
    E2 忘记在循环体内写 cin >> x 循环变量不变，死循环或读不到数据 循环体内必须有读入语句
    E3 哨兵值被计入结果 多算了一个 0（求和时不易发现，求乘积或计数时出错） 用 while (cin>>x && x!=0)，或 if (x==0) break
    E4 for 循环 i < n 写成 i <= n 多读一个数 从 0 开始用 i < n，从 1 开始用 i <= n
    E5 for 循环 i = 0 配 i <= n 读 n+1 次，多读一个 0 开始配 <，1 开始配 <=
    E6 先读 n 却忘了写 cin >> n n 是垃圾值，循环次数错误 先 cin >> n 再 for
    E7 while(cin>>x) 中 x 类型与数据不匹配 读到非数字立即失败，循环提前退出 x 的类型要和题目数据一致
    E8 把 cin >> x 写成 cin >> x; 在 while 条件后多分号 while 条件为空语句，逻辑错误 while 条件后不加分号
    E9 多变量读入写成 while (cin >> a, b) 逗号表达式，b 没被读入 写成 while (cin >> a >> b)
    E10 哨兵判断用 = 而非 == x = 0 是赋值，条件恒假或恒真 用 x != 0，判断用 ==

    累加器初始化位置

    最常见的错误——累加器写在循环体内：

    // 错误：sum = 0 在循环体内，每轮清零
for (int i = 0; i < n; i++) {
    int sum = 0;       // 每轮都变回 0！
    int x;
    cin >> x;
    sum += x;
}
cout << sum;           // 而且 sum 在循环内定义，循环外用不了


    // 正确：sum = 0 在循环外，只初始化一次
int sum = 0;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    sum += x;
}
cout << sum;

    铁律： 累加器（sum）、累乘器（product）、计数器（count）一律在循环外初始化。循环内只做累加/累乘/计数，不做初始化。

    哨兵值混入结果

    // 错误：哨兵值 0 被加进 sum（求和时碰巧不影响，但求乘积就出错）
int sum = 0, x;
while (cin >> x) {
    sum += x;          // 0 也会被加进去
}


    // 正确：用条件排除哨兵值
int sum = 0, x;
while (cin >> x && x != 0) {   // x==0 时退出，0 不被加
    sum += x;
}

    习惯： 哨兵值是"结束标记"，不是数据，永远不参与计算。用 && x != 哨兵值 或 if (x == 哨兵值) break 排除它。

    十三、方法、套路与策略小结

    1. 循环读入三步套路

    • 第 1 步 看题判断场景： 题目给了 n 吗？给了哨兵值吗？都没有吗？→ 决定用 for / while+哨兵 / while(cin>>a)。
    • 第 2 步 初始化累加器： 在循环外把 sum、product、count 初始化好（累加 0、累乘 1、计数 0）。
    • 第 3 步 写循环体： 读入 → 累加/累乘/计数 →（注意哨兵值排除）。

    2. 场景识别决策

    题目关键词 场景 循环写法
    "第一行 n""有 n 个" 指定个数 for (i=0; i<n; i++)
    "直到 0 为止""以 -1 结束" 哨兵终止 while (cin>>x && x!=哨兵)
    "若干个""多组"（无个数无哨兵） 读到 EOF while (cin >> x)
    "每行两个""若干行"（无个数） 多变量 EOF while (cin >> a >> b)

    3. 循环读入经典套路

    套路 模式 典型题目
    指定个数累加 for(i=0;i<n;i++){cin>>x; sum+=x;} 求 n 个数和、求均值
    指定个数多累加器 for(...){cin>>g>>s>>b; gold+=g;...} 奥运奖牌计数
    哨兵终止累加 while(cin>>x && x!=0){sum+=x;} 读到 0 为止求和
    EOF 累加 while(cin>>x){sum+=x;} 若干数求和
    多变量边读边处理 while(cin>>a>>b){cout<<a+b;} 多对 a+b

    4. 信奥策略意识

    • 场景识别： 拿到题目先看输入格式——给个数、给哨兵、还是啥都不给，决定循环写法。
    • 累加器位置： 一律循环外初始化，这是循环读入第一铁律。
    • 哨兵不参与： 哨兵值是结束标记，不是数据，用条件排除。
    • EOF 测试： 本地测试 while(cin>>a) 时，用 Ctrl+Z（Windows）或 Ctrl+D（Linux/Mac）模拟输入结束。
    • 性能意识： 循环内频繁输出用 '\n' 而非 endl。
    • 样例验证： 手动追踪累加器的变化，确认读入次数和哨兵值处理正确。

    附录 · 本课代码模板速查

    模板 1：指定个数读入（for 循环）

    #include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;                       // 先读个数
    int sum = 0;                    // 累加器在循环外初始化
    for (int i = 0; i < n; i++) {   // 循环 n 次
        int x;
        cin >> x;                   // 每次读一个
        sum += x;                   // 累加
    }
    cout << sum << '\n';
    return 0;
}

    适合"第一行 n，后跟 n 个数"的题目。i 从 0 开始用 i < n，从 1 开始用 i <= n。

    模板 2：哨兵终止读入（while 循环）

    #include <iostream>
using namespace std;
int main() {
    int sum = 0;
    int x;
    while (cin >> x && x != 0) {    // 读 x，且 x 不是哨兵值才继续
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    适合"读到 0 为止"的题目。哨兵值（这里是 0）不会被计入结果。换哨兵值改 x != 0 为 x != -1 等。

    模板 3：读到 EOF（while(cin>>a)）

    #include <iostream>
using namespace std;
int main() {
    int sum = 0;
    int x;
    while (cin >> x) {              // 不断读，读到 EOF 就停
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    适合"输入若干个数"（没说几个、没给哨兵）的题目。本地测试用 Ctrl+Z/Ctrl+D 模拟 EOF。

    模板 4：多变量 EOF 读入

    #include <iostream>
using namespace std;
int main() {
    int a, b;
    while (cin >> a >> b) {         // 每次读一对
        cout << a + b << '\n';      // 边读边处理
    }
    return 0;
}

    适合"每行两个数，输入若干行"的题目。每读一对就处理一对。

    模板 5：指定个数 + 多累加器

    #include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int gold = 0, silver = 0, bronze = 0;   // 多个累加器都在循环外
    for (int i = 0; i < n; i++) {
        int g, s, b;
        cin >> g >> s >> b;                 // 一行读多个
        gold += g;
        silver += s;
        bronze += b;
    }
    cout << gold << ' ' << silver << ' ' << bronze << '\n';
    return 0;
}

    适合"每行多个数据，分别累计"的题目（如奥运奖牌计数）。多个累加器都在循环外初始化。

    模板 6：哨兵终止 + break 写法

    #include <iostream>
using namespace std;
int main() {
    int sum = 0;
    int x;
    while (cin >> x) {              // 一直读
        if (x == 0) break;         // 读到哨兵值就跳出
        sum += x;
    }
    cout << sum << '\n';
    return 0;
}

    与模板 2 等价的另一种写法，用 break 跳出。逻辑更直白，初学者易理解。
