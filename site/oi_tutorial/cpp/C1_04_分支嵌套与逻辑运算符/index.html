    第 4 课 · 多条件分支与逻辑运算符

    逻辑运算符 · 短路求值 · 组合条件 · 嵌套 if · 悬空 else · 分级题单

    一、教学目标

    本课目标依据布鲁姆教育目标分类学（修订版）制定，采用 ABCD 撰写法（主体 A、行为动词 B、行为条件 C、表现程度 D），确保目标可观测、可评估。

    1. 课时目标（ABCD 表述）

    编号 目标表述（ABCD） 分类定位
    G1 记忆 在给出 3 个逻辑运算符的条件下，学生能够准确写出 &&、\|\|、! 的含义与读法，无遗漏。 事实性知识 × 记忆
    G2 理解 在给出 &&、\|\| 真值表的条件下，学生能够解释"两边都为真结果才为真""至少一个为真结果就为真"，并说明短路求值。 概念性知识 × 理解
    G3 应用 在无参考的条件下，学生能够用 &&、\|\|、! 正确组合多个比较条件，编写多条件分支程序，语法正确率不低于 90%。 程序性知识 × 应用
    G4 应用 在给出需要"先分大类再分小类"的问题的条件下，学生能够正确编写嵌套 if 结构，并避免悬空 else 错误。 程序性知识 × 应用
    G5 分析 在给出含错误代码的条件下，学生能够区分"&& 写成 &""悬空 else 配对错误""浮点数用 == 比较"等典型错误并改正。 程序性知识 × 分析

    2. 目标说明

    • 认知层次： 本课承接第 3 课，认知重心在"理解—应用"，分析层以纠错形式渗透。
    • 知识类型： 概念性知识（逻辑组合、真/假、短路）与程序性知识（逻辑运算符、嵌套 if）并重。
    • 边界： 本课在第 3 课"单条件分支"基础上，学习"多条件组合"的两种方式——逻辑运算符连接、嵌套 if。学完本课可完成含闰年判断、三角形判断、范围判断等多条件分支题目。循环结构（for/while）留待后续课程。

    三、教学重点与难点

    1. 教学重点

    • 3 个逻辑运算符 &&（且）、||（或）、!（非）的含义与组合用法。
    • 真值表的理解：&& 两边都真才真，|| 至少一真就真。
    • 短路求值：&& 左边为假时右边不计算，|| 左边为真时右边不计算。
    • 嵌套 if 的写法：if 里套 if，用于"先分大类再分小类"。
    • 用逻辑运算符组合条件解决实际问题（闰年、三角形、范围判断）。

    2. 教学难点

    • && 与 || 的混淆——"且"还是"或"要想清楚，写反逻辑全错。
    • && 写成 &（单个与号）——变成位运算，结果可能错。
    • 悬空 else：else 总是与最近的未配对 if 配对，缩进会骗人。
    • 浮点数不能用 == 直接比较（精度误差），要用误差范围判断。
    • 逻辑运算符优先级：! > && > ||，拿不准就加括号。

    四、为什么需要多条件

    1. 第 3 课的局限

    第 3 课学的 if，每个条件都是单个比较——比如 score >= 60、n % 2 == 0。一个条件，一次判断。

    但很多问题需要多个条件同时满足，或者多个条件满足其一：

    • 判断闰年：能被 4 整除且不能被 100 整除，或者能被 400 整除。
    • 判断两位数：n ≥ 10 且 n ≤ 99。
    • 判断三角形：a+b>c 且 a+c>b 且 b+c>a。
    • 判断字母：(ch ≥ 'a' 且 ch ≤ 'z') 或者 (ch ≥ 'A' 且 ch ≤ 'Z')。

    这些都需要把多个条件组合起来——这就是本课的核心。

    2. 两种组合方式

    C++ 提供两种方式实现多条件：

    方式 做法 适合场景
    逻辑运算符 用 &&、\|\|、! 把多个条件连成一个表达式 条件间"且""或"关系，条件不复杂
    嵌套 if if 里再套 if，分层判断 "先分大类，再分小类"的层次结构

    两种方式往往可以互相转换，选哪种看哪种更清晰。本课先学逻辑运算符，再学嵌套 if，最后对比选择。

    五、逻辑运算符

    1. 三个逻辑运算符

    运算符 含义 读法 示例
    && 逻辑与（且） "并且" a > 0 && b > 0（a、b 都大于 0 才成立）
    \|\| 逻辑或 "或者" a > 0 \|\| b > 0（a、b 至少一个大于 0 就成立）
    ! 逻辑非 "非" !(a > 0)（a 不大于 0，即 a ≤ 0）

    2. &&（与）：两边都真才真

    int a = 5, b = 3;
cout << (a > 0 && b > 0);    // 5>0 且 3>0，都成立，输出 1（true）
cout << (a > 0 && b < 0);    // 5>0 成立，3<0 不成立，输出 0（false）

    理解要点： && 像"串联开关"——两个开关都闭合，电路才通。

    3. ||（或）：至少一真就真

    int a = 5, b = -3;
cout << (a > 0 || b > 0);    // 5>0 成立（真），输出 1（true）
cout << (a < 0 || b > 0);    // 5<0 不成立，-3>0 不成立，输出 0（false）

    理解要点： || 像"并联开关"——任一个开关闭合，电路就通。

    4. !（非）：取反

    int a = 5;
cout << !(a > 0);     // a>0 成立（真），取反为假，输出 0
cout << !(a < 0);     // a<0 不成立（假），取反为真，输出 1

    !(a > 0) 等价于 a <= 0。能用比较运算符直接写的，尽量直接写，! 多用于复杂条件取反。

    六、真值表

    1. &&（与）真值表

    A B A && B
    真 真 真
    真 假 假
    假 真 假
    假 假 假

    只有两边都为真，结果才为真。

    2. ||（或）真值表

    A B A || B
    真 真 真
    真 假 真
    假 真 真
    假 假 假

    只要至少一个为真，结果就为真。

    3. !（非）真值表

    A !A
    真 假
    假 真

    七、短路求值

    1. 什么是短路求值

    • &&：如果左边为假，右边不再计算（因为结果一定是假）。
    • ||：如果左边为真，右边不再计算（因为结果一定是真）。
    int a = 0;
// a==0 时左边 (a != 0) 为假，右边 10/a 不计算，避免除以 0
if (a != 0 && 10 / a > 1) {
    cout << "安全" << endl;
}

    2. 短路的实际价值

    利用短路避免错误： 把可能出错的判断（如除以 0）放在 && 右边，左边先做安全检查。左边检查不通过（为假），右边就不计算，避免了运行错误。

    // 安全：先检查 b != 0，再算 a/b
if (b != 0 && a / b > 2) { ... }

// 危险：如果 b==0，a/b 会除以 0 导致程序崩溃
if (a / b > 2 && b != 0) { ... }   // 顺序写反了！

    口诀： && 把安全检查放左边，危险操作放右边；|| 把容易成立的放左边，能尽早短路。

    八、用逻辑运算符组合条件（经典例子）

    1. 判断两位数（B2040）

    一个正整数是两位数，当且仅当它 ≥ 10 且 ≤ 99：

    int n;
cin >> n;
if (n >= 10 && n <= 99) {
    cout << 1 << endl;    // 是两位数
} else {
    cout << 0 << endl;    // 不是两位数
}

    第 3 课只能用嵌套 if 分两步判断，现在用 && 一行搞定。

    2. 判断字母

    一个字符是字母，当且仅当它是小写字母（'a'~'z'）**或**大写字母（'A'~'Z'）：

    char ch;
cin >> ch;
if ((ch >= 'a' && ch <= 'z') || (ch >= 'A' && ch <= 'Z')) {
    cout << "yes" << endl;
} else {
    cout << "no" << endl;
}

    括号建议： 两个范围用括号分组，再用 || 连接，清晰不易错。

    3. 闰年判断（P5711）

    闰年规则：能被 4 整除且不能被 100 整除，或者能被 400 整除。

    int year;
cin >> year;
if ((year % 4 == 0 && year % 100 != 0) || (year % 400 == 0)) {
    cout << 1 << endl;    // 闰年
} else {
    cout << 0 << endl;    // 平年
}

    括号建议： 逻辑表达式较复杂时，加括号分组，既不会错，也更好读。这是逻辑运算符组合条件的经典例子。

    4. 三角形判断（B2050）

    三条线段能构成三角形，当且仅当任意两边之和大于第三边：

    int a, b, c;
cin >> a >> b >> c;
if (a + b > c && a + c > b && b + c > a) {
    cout << 1 << endl;    // 能构成三角形
} else {
    cout << 0 << endl;    // 不能
}

    三个条件用 && 连接，三个都成立才是三角形。

    九、嵌套 if 语句

    1. 什么是嵌套 if

    if 里还可以套 if，用于"先分大类，再分小类"的层次判断。

    2. 示例：先判段再判级

    int score;
cin >> score;
if (score >= 60) {
    if (score >= 90) {
        cout << "优秀" << endl;
    } else {
        cout << "及格" << endl;
    }
} else {
    cout << "不及格" << endl;
}

    执行流程： 先判断 score >= 60（大类：及格/不及格），及格后再判断 score >= 90（小类：优秀/及格）。

    3. 嵌套 if 与逻辑运算符的等价转换

    很多嵌套 if 可以用逻辑运算符改写。比如"判断 a 是两位数"：

    // 嵌套 if 写法
if (n >= 10) {
    if (n <= 99) {
        cout << "两位数" << endl;
    }
}

// 逻辑运算符写法（等价，更简洁）
if (n >= 10 && n <= 99) {
    cout << "两位数" << endl;
}

    选择建议： 条件之间是"且"关系且都不复杂时，用 && 更简洁；需要"先分大类再分小类"的层次结构时，用嵌套 if 更清晰。

    4. 嵌套 if 的层数

    嵌套层数不宜过多。超过 3 层时考虑用逻辑运算符或调整判断顺序简化。层数太多代码难读、容易出错。

    十、悬空 else 陷阱

    1. 问题代码

    if (a > 0)
    if (b > 0)
        cout << "A";
else
    cout << "B";

    缩进看起来 else 配对第一个 if，但实际上 else 配对最近的 if（第二个）。等价于：

    if (a > 0) {
    if (b > 0)
        cout << "A";
    else
        cout << "B";      // else 配的是第二个 if！
}

    2. 配对规则

    规则： else 总是与最近的、未配对的 if 配对，与缩进无关。

    3. 避免方法

    加花括号，明确配对关系：

    // 想让 else 配第一个 if，加花括号
if (a > 0) {
    if (b > 0)
        cout << "A";
} else {
    cout << "B";          // 现在 else 配第一个 if
}

    规范意识： 嵌套 if 时一律加花括号，这是避免悬空 else 最有效的办法。缩进会骗人，花括号不会。

    十一、两种多条件实现方式的对比与选择

    对比项 逻辑运算符 嵌套 if
    写法 if (a > 0 && b > 0) if 里套 if
    适合场景 条件间"且""或"关系 "先分大类再分小类"层次结构
    可读性 条件少时简洁 条件多时层次清晰
    易错点 && 与 \|\| 混淆、优先级 悬空 else、配对错误
    互换性 多数情况可互相转换 多数情况可互相转换

    选择决策

    • 条件之间是"且""或"关系，且每个条件都不复杂 → 用逻辑运算符。
    • 需要"先分大类再分小类"，大类和小类判断不同 → 用嵌套 if。
    • 条件超过 3 个组合 → 考虑拆成嵌套 if 或调整逻辑，避免一个 if 里条件太长。
    • 无论哪种方式，一律加花括号。

    十二、三目运算符（条件表达式，拓展）

    1. 语法

    条件 ? 表达式1 : 表达式2

    含义： 条件为真，整个表达式的值是"表达式1"；为假，值是"表达式2"。

    2. 示例：两数较大值

    int a, b;
cin >> a >> b;
int mx = (a >= b) ? a : b;   // a>=b 时 mx=a，否则 mx=b
cout << mx << endl;

    等价于：

    int mx;
if (a >= b) mx = a;
else mx = b;

    建议： 三目运算符适合"二选一取值"的简短场景，逻辑复杂时用 if-else 更清晰，别为短而短。

    十三、switch 语句（拓展，了解即可）

    switch 用于"一个变量等于不同值时走不同分支"，适合等值判断（不能判断大小范围）。

    1. 语法

    switch (表达式) {
    case 值1:
        语句1;
        break;
    case 值2:
        语句2;
        break;
    default:
        默认语句;
        break;
}

    2. 示例：星期几

    int day;
cin >> day;
switch (day) {
    case 1: cout << "星期一" << endl; break;
    case 2: cout << "星期二" << endl; break;
    case 3: cout << "星期三" << endl; break;
    // ...
    default: cout << "输入有误" << endl; break;
}

    3. 两个要点

    • break 不能漏： 漏了 break 会"穿透"（fall-through），继续执行下一个 case 的语句，直到遇到 break 或 switch 结束。
    • 表达式只能是整型/字符型： 不能用 double、string，也不能写 case score >= 60: 这种范围。

    信奥建议： 入门阶段分支题用 if-else if-else 足够，switch 用得少，了解即可。判断范围用 if，判断离散等值可用 switch。

    十四、常见错误与注意事项

    编号 错误 后果 改正
    E1 && 写成 &（单个与号） 变成位运算，结果可能错 逻辑与用两个 &&
    E2 \|\| 写成 \|（单个或号） 变成位运算，结果可能错 逻辑或用两个 \|\|
    E3 && 与 \|\| 混淆 "且"写成"或"或反之，逻辑全错 想清楚是"都满足"还是"满足其一"
    E4 悬空 else 配对错误 else 配了非预期的 if 加花括号明确配对
    E5 浮点数用 == 比较 精度误差，可能永远不等 用误差范围判断
    E6 比较运算符连写（a < b < c） 先算 a<b 得 0/1，再和 c 比，逻辑错 用 a < b && b < c
    E7 逻辑运算符优先级记错 ! > && > \|\|，不加括号可能错 复杂条件加括号
    E8 嵌套层数过多 代码难读、易错 超过 3 层考虑简化
    E9 短路求值顺序写反 危险操作放左边，可能除以 0 安全检查放 && 左边
    E10 switch 漏 break 穿透执行后续 case 每个 case 末尾加 break

    浮点数比较的正确写法

    // 错误：精度误差，0.1+0.2 可能不精确等于 0.3
if (x == 0.3) { ... }

// 正确：用误差范围
if (fabs(x - 0.3) < 1e-6) { ... }   // 差值小于 0.000001 就认为相等

    fabs 需要 #include <cmath>。浮点数存储有精度误差，直接用 == 比较可能永远不相等。用"差值足够小"来判断才是正确做法。

    逻辑运算符优先级

    !（非）  >  &&（且）  >  ||（或）

    所以 a || b && c 等价于 a || (b && c)。拿不准就加括号，括号既不会错，也更好读。

    输入法提醒

    • 写代码一律英文输入法。&&、||、! 都是英文符号，中文符号会报错。

    十五、方法、套路与策略小结

    1. 多条件分支两步套路

    • 第1步 分析条件关系： 想清楚多个条件之间是"且"（都满足）还是"或"（满足其一）关系。
    • 第2步 选择实现方式： "且/或"关系用逻辑运算符，"先分大类再分小类"用嵌套 if。

    2. 条件书写决策

    • 单个条件 → 直接用比较运算符（第 3 课）。
    • 多个条件"同时成立" → 用 && 连接。
    • 多个条件"至少一个成立" → 用 || 连接。
    • 取反 → 用 !，或改写条件（!(a > b) 等价于 a <= b）。
    • 范围判断 → a >= 10 && a <= 99，不能写成 10 <= a <= 99。
    • 复杂条件 → 加括号分组，不靠记忆优先级。

    3. 多条件实现选择决策

    场景 用法
    条件间"且""或"关系，条件不复杂 逻辑运算符 &&、\|\|
    先分大类再分小类 嵌套 if
    二选一取值赋给变量 三目运算符（拓展）
    离散等值（星期几、运算符） switch（拓展）

    4. 信奥策略意识

    • 想清楚"且"还是"或"： 写条件前先想清楚是"都满足"还是"满足其一"，写完回头检查 && 和 || 有没有用反。
    • 复杂条件加括号： 逻辑运算符组合时用括号分组，不靠记忆优先级。
    • 一律加花括号： 嵌套 if 时尤其重要，杜绝悬空 else。
    • 短路求值利用： 安全检查放 && 左边，危险操作放右边。
    • 样例验证： 多条件题要测每种条件组合——比如闰年要测 2000（闰）、1900（平）、2004（闰）、2001（平）四种情况。
    • 边界必测： 范围判断测边界值，如判断两位数测 9、10、99、100。

    自编-1： 读入一个字符，判断它是否是字母（a-z 或 A-Z）。是字母输出 yes，否则输出 no。

    输入: F
输出: yes

    提示：(ch >= 'a' && ch <= 'z') || (ch >= 'A' && ch <= 'Z')。

    输入: 1 -5 6

    提示：Δ = b_b - 4_a*c，用 if-else if-else 三分支。Δ 是整数可直接用 == 比较。

输出: yes

    附录 · 本课代码模板速查

    模板 1：逻辑运算符速记

    // 逻辑与（且）：两边都真才真
a > 0 && b > 0        // a、b 都大于 0

// 逻辑或：至少一真就真
a > 0 || b > 0        // a、b 至少一个大于 0

// 逻辑非：取反
!(a > 0)              // 等价于 a <= 0

// 范围判断（必须用 &&）
a >= 10 && a <= 99    // a 在 10~99 之间

    模板 2：多条件分支万能框架

    #include <iostream>
using namespace std;
int main(){
    int n;
    cin >> n;
    if (n >= 10 && n <= 99) {        // 多条件用 && 组合
        cout << "两位数" << endl;
    } else {
        cout << "不是两位数" << endl;
    }
    return 0;
}

    模板 3：闰年判断（逻辑组合经典）

    int year;
cin >> year;
if ((year % 4 == 0 && year % 100 != 0) || (year % 400 == 0)) {
    cout << 1 << endl;   // 闰年
} else {
    cout << 0 << endl;   // 平年
}

    模板 4：三角形判断（多条件 && 组合）

    int a, b, c;
cin >> a >> b >> c;
if (a + b > c && a + c > b && b + c > a) {
    cout << 1 << endl;   // 能构成三角形
} else {
    cout << 0 << endl;   // 不能
}

    模板 5：嵌套 if 标准框架

    if (大类条件) {
    if (小类条件) {
        // 大类成立 且 小类成立
    } else {
        // 大类成立 且 小类不成立
    }
} else {
    // 大类不成立
}

    一律加花括号，避免悬空 else。嵌套层数不宜超过 3 层，超过时考虑用逻辑运算符简化。

    模板 6：浮点数比较正确写法

    #include <cmath>
double x;
cin >> x;
if (fabs(x - 0.3) < 1e-6) {     // 差值小于 0.000001 就认为相等
    cout << "相等" << endl;
} else {
    cout << "不相等" << endl;
}

    浮点数不能用 == 直接比较，用"差值足够小"判断。
